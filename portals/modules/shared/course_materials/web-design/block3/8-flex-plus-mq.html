Putting It All Together: Combining Flexbox layouts with Media Queries to build a complete responsive webpage.

So far, you have learned two powerful tools:

  1. Flexbox: For creating flexible one-dimensional layouts.
  2. Media Queries: For applying CSS styles based on screen size.

On their own, they are useful, but their true power is unlocked when they are combined. Media Queries allow you to change the rules of your Flexbox layout at different screen sizes, creating a seamless experience on any device.

The Core Concept: Responsive Flexbox Layouts

The goal is to write a "mobile-first" base style (usually a simple, stacked layout) and then use Media Queries to overwrite specific Flexbox properties for larger screens where more space is available.

You are not choosing between Flexbox and Media Queries; you are using Media Queries to adapt your Flexbox code.

Scenario 1: Adapting a Navigation Bar

A common pattern is a navigation bar that is stacked vertically on a phone and horizontal on a desktop.

The HTML:

<nav class="navbar">
    <a href="#" class="logo">MySite</a>
    <ul class="nav-links">
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Services</a></li>
        <li><a href="#">Contact</a></li>
    </ul>
</nav>

The CSS (Mobile First - Stacked):

.navbar {
    display: flex;
    flex-direction: column; /* Stacks logo and links vertically */
    align-items: center;    /* Centers items in the column */
}

.nav-links {
    display: flex;
    flex-direction: column; /* Stacks the list items vertically */
    align-items: center;
    list-style: none;
    width: 100%;
}

The CSS (Larger Screens - Horizontal):

/* When the screen is 600px wide or larger... */
@media (min-width: 600px) {
    .navbar {
        flex-direction: row;      /* Changes main axis to horizontal */
        justify-content: space-between; /* Pushes logo and links apart */
        align-items: center;
    }

    .nav-links {
        flex-direction: row;      /* Makes the list items horizontal */
        justify-content: flex-end; /* Aligns links to the right */
        width: auto;              /* Allows the nav to shrink to fit content */
    }
    
    .nav-links li {
        margin-left: 20px; /* Adds space between horizontal links */
    }
}

Scenario 2: Adapting a Card Gallery

A gallery of cards might show one card per row on a phone, two on a tablet, and three or four on a desktop.

The HTML:

<section class="card-gallery">
    <div class="card">Card 1 Content</div>
    <div class="card">Card 2 Content</div>
    <div class="card">Card 3 Content</div>
    <div class="card">Card 4 Content</div>
</section>

The CSS (Mobile First - Single Column):

.card-gallery {
    display: flex;
    flex-direction: column; /* Stacks cards vertically */
    gap: 20px; /* Adds space between stacked cards */
}

.card {
    /* Card styling (background, border, padding) */
    background-color: #f4f4f4;
    padding: 1rem;
}

The CSS (Larger Screens - Multi-Column):

/* For tablets: 2 columns */
@media (min-width: 600px) {
    .card-gallery {
        flex-direction: row;      /* Allows items to sit side-by-side */
        flex-wrap: wrap;          /* Allows items to wrap to the next line */
    }

    .card {
        flex-basis: calc(50% - 10px); /* Each card takes roughly half the width minus half the gap */
    }
}

/* For desktops: 3 columns */
@media (min-width: 900px) {
    .card {
        flex-basis: calc(33.333% - 14px); /* Each card takes roughly one-third of the width */
    }
}

Key Takeaways for Combining Flexbox & Media Queries

  1. Start Mobile-First: Write your base CSS for the smallest screen (usually a single column, stacked layout). This is simpler and forces you to prioritize content.
  2. Use min-width Breakpoints: Using min-width (applies styles above a certain width) aligns perfectly with a mobile-first strategy.
  3. Only Override What's Necessary: Inside your media query, you only need to rewrite the Flexbox properties that change. You don't need to repeat display: flex; if it's already in your base styles.
  4. Change the flex-direction: This is the most common and powerful change. Switching from column to row is the foundation of most responsive layout shifts.
  5. Adjust Sizing with flex-basis: When items wrap, use flex-basis (often within a media query) to control how much space each item takes up on different screen sizes.

By combining these two techniques, you move from building pages that simply "shrink" to building pages that re-structure themselves to provide the optimal user experience on any screen.
