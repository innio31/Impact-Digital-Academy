JAVASCRIPT FUNDAMENTALS: INTRODUCTION TO JAVASCRIPT & VARIABLES

A COMPREHENSIVE GUIDE

PART 1: INTRODUCTION TO JAVASCRIPT

1.1 What is JavaScript?

JavaScript (often abbreviated as JS) is a high-level, interpreted programming language that is one of the core technologies of the World Wide Web, alongside HTML and CSS. It was created by Brendan Eich in 1995 while he was working at Netscape Communications Corporation. The language was originally named Mocha, then briefly LiveScript, before finally being renamed JavaScript.

Key Characteristics of JavaScript:

Characteristic Description
High-level Abstracted from low-level computer operations, making it easier to write
Interpreted Code is executed line-by-line without prior compilation
Dynamic Types are associated with values, not variables
Weakly typed Implicit type conversion occurs when operations involve mismatched types
Prototype-based Uses prototypes for inheritance rather than classes
Single-threaded Executes one command at a time
Non-blocking Uses asynchronous operations to handle multiple tasks

1.2 The Role of JavaScript in Web Development

JavaScript serves three primary roles in modern web development:

A. Front-End Development (Client-Side)

JavaScript runs in the user's browser and is responsible for:

1. Dynamic Content Manipulation
// Example: Changing content dynamically
document.getElementById("welcome-message").textContent = "Hello, Welcome to our site!";
2. User Interaction and Event Handling
// Example: Responding to user clicks
button.addEventListener("click", function() {
    alert("Button was clicked!");
});
3. Form Validation
// Example: Validating form input before submission
function validateEmail(email) {
    if (!email.includes("@")) {
        alert("Please enter a valid email address");
        return false;
    }
    return true;
}
4. Animations and Visual Effects
// Example: Simple fade-in effect
element.style.transition = "opacity 1s";
element.style.opacity = "1";
5. AJAX and API Communication
// Example: Fetching data from an API
fetch("https://api.example.com/users")
    .then(response => response.json())
    .then(data => displayUsers(data));

B. Back-End Development (Server-Side)

With the introduction of Node.js in 2009, JavaScript can now run on servers:

1. Building Web Servers
// Example: Basic HTTP server with Node.js
const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, {'Content-Type': 'text/plain'});
    res.end('Hello World from Server!');
});

server.listen(3000);
2. Database Operations
// Example: Connecting to MongoDB
const mongoose = require('mongoose');
mongoose.connect('mongodb://localhost/myapp');
3. File System Operations
// Example: Reading a file
const fs = require('fs');
fs.readFile('document.txt', 'utf8', (err, data) => {
    console.log(data);
});

C. Full-Stack Development

JavaScript can be used for both front-end and back-end development, making it a full-stack language. This concept is often referred to as "JavaScript Everywhere" or "Isomorphic JavaScript."

PART 2: ADDING JAVASCRIPT TO A WEB PAGE

2.1 The <script> Tag

The <script> tag is the primary method for including JavaScript in HTML documents. It can be used in three different ways:

Method 1: Inline JavaScript

<!DOCTYPE html>
<html>
<head>
    <title>Inline JavaScript Example</title>
</head>
<body>
    <h1>My Web Page</h1>
    
    <script>
        // This is inline JavaScript
        console.log("Hello from inline JavaScript!");
        alert("Welcome to my website!");
        
        let userName = "John";
        document.write("<p>Hello, " + userName + "!</p>");
    </script>
</body>
</html>

Method 2: External JavaScript File

script.js:

// This is an external JavaScript file
console.log("Hello from external JavaScript!");

function greetUser(name) {
    return `Hello, ${name}!`;
}

document.addEventListener("DOMContentLoaded", function() {
    const button = document.getElementById("myButton");
    button.addEventListener("click", function() {
        alert("Button was clicked!");
    });
});

index.html:

<!DOCTYPE html>
<html>
<head>
    <title>External JavaScript Example</title>
    <!-- Linking external JavaScript file -->
    <script src="script.js"></script>
</head>
<body>
    <h1>My Web Page</h1>
    <button id="myButton">Click Me!</button>
</body>
</html>

Method 3: Inline Event Handlers (Not Recommended)

<!DOCTYPE html>
<html>
<body>
    <!-- NOT RECOMMENDED - Old style -->
    <button onclick="alert('Button clicked!'); return false;">
        Click Me
    </button>
    
    <a href="javascript:void(0)" onclick="doSomething()">
        Click Here
    </a>
    
    <body onload="initializePage()">
</body>
</html>

2.2 Placement of <script> Tags

Option A: <script> in <head>

<!DOCTYPE html>
<html>
<head>
    <title>Script in Head</title>
    <script src="script.js"></script>
</head>
<body>
    <h1>My Page</h1>
</body>
</html>

Option B: <script> at the end of <body>

<!DOCTYPE html>
<html>
<head>
    <title>Script at End of Body</title>
</head>
<body>
    <h1>My Page</h1>
    <p>Some content...</p>
    
    <script src="script.js"></script>
</body>
</html>

Option C: Using async and defer attributes

<!DOCTYPE html>
<html>
<head>
    <title>Async and Defer Examples</title>
    
    <!-- Defer: Execute after HTML parsing, in order -->
    <script defer src="script1.js"></script>
    <script defer src="script2.js"></script>
    
    <!-- Async: Execute as soon as downloaded (not in order) -->
    <script async src="analytics.js"></script>
    
    <!-- Combined (rare) -->
    <script async defer src="modern.js"></script>
</head>
<body>
    <h1>My Page</h1>
    <p>Content...</p>
</body>
</html>

Comparison of Script Loading Methods:

Method HTML Parsing Script Execution Order Preserved Use Case
In <head> Blocked Immediate after download Yes Critical scripts needed before rendering
End of <body> Complete After full HTML parse Yes Most general scripts
defer Continues After HTML parse Yes Scripts that need full DOM
async Continues Immediate after download No Independent scripts (analytics, ads)

PART 3: VARIABLES IN JAVASCRIPT

3.1 What are Variables?

Variables are containers for storing data values. Think of them as labeled boxes where you can put information and retrieve it later using the label.

Analogy: The Storage Box System

┌─────────────────────┐
│      Variable       │
│    ┌───────────┐    │
│    │  Value    │    │
│    │  "John"   │    │
│    └───────────┘    │
│      Label:         │
│    "userName"       │
└─────────────────────┘

3.2 Variable Declaration in JavaScript

JavaScript provides three keywords for declaring variables: let, const, and var (though var is outdated).

The Evolution: var vs let vs const

// OLD WAY (pre-2015) - Avoid using 'var' in modern code
var oldWay = "This is old style";

// MODERN WAY (ES6+) - Use 'let' and 'const'
let modernWay = "This is modern";
const PI = 3.14159;

3.2.1 The let Keyword

// Basic let declaration
let message;
console.log(message); // undefined (variable exists but has no value)

// Assigning a value
message = "Hello World";
console.log(message); // "Hello World"

// Reassigning a value
message = "New Message";
console.log(message); // "New Message"

// Declaring and assigning simultaneously
let count = 10;
console.log(count); // 10

// Multiple declarations
let x = 5, y = 10, z = 15;
console.log(x, y, z); // 5 10 15

// Real-world examples with let
let userName = "John";
let userAge = 25;
let isLoggedIn = false;

// Variables that change over time
let score = 0;
score = score + 10; // score becomes 10
score += 5;         // score becomes 15
score++;            // score becomes 16

let currentTemp = 72;
currentTemp = 68;   // Temperature changed

3.2.2 The const Keyword

// Basic const declaration (must be initialized)
const PI = 3.14159;
console.log(PI); // 3.14159

// THIS WILL CAUSE AN ERROR:
// PI = 3.14; // TypeError: Assignment to constant variable

// Must initialize when declaring
// const wrong; // SyntaxError: Missing initializer in const declaration

// Real-world constants
const COMPANY_NAME = "Tech Corp";
const MAX_USERS = 100;
const API_KEY = "abc123xyz789";
const BIRTH_YEAR = 1990;

// const with objects (reference remains constant, properties can change)
const person = {
    name: "Alice",
    age: 30
};

// This is allowed (modifying object property)
person.age = 31;
console.log(person); // { name: "Alice", age: 31 }

// This is NOT allowed (reassigning the object)
// person = { name: "Bob" }; // TypeError

// const with arrays
const colors = ["red", "green", "blue"];
colors.push("yellow");     // Allowed
colors[0] = "orange";      // Allowed
console.log(colors);       // ["orange", "green", "blue", "yellow"]

// This is NOT allowed
// colors = ["black", "white"]; // TypeError

3.2.3 The var Keyword (Legacy)

// var declaration (avoid in modern code)
var oldVariable = "This is old style";

// var is function-scoped, not block-scoped
if (true) {
    var test = "I'm using var";
}
console.log(test); // "I'm using var" (accessible outside the block!)

// var can be redeclared (can cause bugs)
var name = "John";
var name = "Jane"; // No error!
console.log(name); // "Jane"

// var is hoisted (moved to top of function)
console.log(hoistedVar); // undefined (not an error!)
var hoistedVar = "This is hoisted";

// This is equivalent to:
// var hoistedVar;
// console.log(hoistedVar);
// hoistedVar = "This is hoisted";

Comparison Table: let vs const vs var

Feature let const var
Reassignable ✅ Yes ❌ No ✅ Yes
Redeclarable ❌ No ❌ No ✅ Yes
Block Scoped ✅ Yes ✅ Yes ❌ No (function scoped)
Hoisted ✅ Yes (but not initialized) ✅ Yes (but not initialized) ✅ Yes (initialized as undefined)
Must initialize ❌ No ✅ Yes ❌ No
Global Property ❌ No ❌ No ✅ Yes (becomes window property)
When to use When value changes When value shouldn't change Avoid in modern code

3.3 Variable Naming Rules and Conventions

Rules (Must Follow):

// ✅ VALID variable names
let name = "John";
let _private = "secret";
let $dollar = "currency";
let firstName = "John";
let user123 = "data";
let _ = "underscore";
let $ = "dollar sign";

// ❌ INVALID variable names
// let 123user = "test";     // Cannot start with number
// let my-name = "John";      // Hyphen not allowed
// let my name = "John";      // Space not allowed
// let class = "math";        // Reserved keyword
// let let = "variable";      // Cannot use keyword as name

// Reserved keywords (cannot use as variable names)
// break, case, catch, class, const, continue, debugger, default,
// delete, do, else, export, extends, finally, for, function, if,
// import, in, instanceof, new, return, super, switch, this, throw,
// try, typeof, var, void, while, with, yield

Conventions (Best Practices):

// 1. Use camelCase for variables (standard in JavaScript)
let firstName = "John";
let lastName = "Doe";
let dateOfBirth = "1990-01-01";
let isLoggedIn = false;

// 2. Use UPPER_SNAKE_CASE for true constants
const MAX_RETRY_COUNT = 3;
const API_BASE_URL = "https://api.example.com";
const DEFAULT_COLOR = "#000000";

// 3. Use descriptive, meaningful names
// ❌ Bad
let d = new Date();
let u = getUser();
let x = 30;

// ✅ Good
let currentDate = new Date();
let currentUser = getUser();
let maxAge = 30;

// 4. Boolean variables should sound like yes/no questions
let isActive = true;
let hasPermission = false;
let canEdit = true;
let shouldDisplay = false;

// 5. Use nouns for variables (things), verbs for functions (actions)
let user = { name: "John" };        // noun
function getUser() { }               // verb
let totalPrice = 100;                // noun
function calculateTotal() { }        // verb

3.4 Variable Scope

Global Scope

// Global variable
let globalVar = "I'm global";

function testFunction() {
    console.log(globalVar); // Accessible: "I'm global"
}

if (true) {
    console.log(globalVar); // Accessible: "I'm global"
}

console.log(globalVar); // Accessible: "I'm global"

Function Scope

function myFunction() {
    let functionVar = "I'm inside function";
    console.log(functionVar); // Works: "I'm inside function"
}

myFunction();
// console.log(functionVar); // ReferenceError: functionVar is not defined

Block Scope

if (true) {
    let blockVar = "I'm inside block";
    const alsoBlockVar = "Me too";
    console.log(blockVar);     // Works
    console.log(alsoBlockVar); // Works
}
// console.log(blockVar);     // ReferenceError
// console.log(alsoBlockVar); // ReferenceError

// Loop block scope
for (let i = 0; i < 3; i++) {
    // i is accessible here
    console.log(i);
}
// console.log(i); // ReferenceError: i is not defined

Scope Chain

let global = "global";

function outerFunction() {
    let outer = "outer";
    
    function innerFunction() {
        let inner = "inner";
        console.log(inner);  // Finds in current scope
        console.log(outer);  // Finds in outer scope
        console.log(global); // Finds in global scope
    }
    
    innerFunction();
    // console.log(inner); // ReferenceError: inner is not defined
}

outerFunction();

4.1 Understanding Data Types

JavaScript has 8 data types, divided into two categories: Primitive and Non-Primitive (Reference).

Key Difference: Primitive vs Reference

// PRIMITIVE TYPES - Stored by value
let a = 10;
let b = a;  // b gets a COPY of the value 10
a = 20;
console.log(a); // 20
console.log(b); // 10 (unchanged)

// REFERENCE TYPES - Stored by reference
let obj1 = { name: "John" };
let obj2 = obj1;  // obj2 points to the SAME object in memory
obj1.name = "Jane";
console.log(obj1.name); // "Jane"
console.log(obj2.name); // "Jane" (both changed)

4.2 The Number Type

// Integer numbers
let integer = 42;
let negative = -10;
let zero = 0;

// Floating point numbers (decimals)
let decimal = 3.14;
let scientific = 1.5e6;  // 1.5 * 10^6 = 1,500,000
let tiny = 1.2e-5;        // 0.000012

// Special numeric values
let infinity = Infinity;
let negInfinity = -Infinity;
let notANumber = NaN;      // Not a Number

// Examples of NaN
console.log(0 / 0);        // NaN
console.log(parseInt("abc")); // NaN
console.log(Math.sqrt(-1)); // NaN

// Number operations
let sum = 10 + 5;          // 15
let difference = 10 - 5;   // 5
let product = 10 * 5;      // 50
let quotient = 10 / 5;     // 2
let remainder = 10 % 3;    // 1 (modulo)
let power = 2 ** 3;        // 8 (exponentiation)

// Number precision issues (IEEE 754 floating point)
console.log(0.1 + 0.2);    // 0.30000000000000004 (!)
// Solution: Round to desired precision
let result = (0.1 + 0.2).toFixed(2); // "0.30"

// Number limits
console.log(Number.MAX_VALUE);      // 1.7976931348623157e+308
console.log(Number.MIN_VALUE);      // 5e-324
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991
console.log(Number.MIN_SAFE_INTEGER); // -9007199254740991

// Checking if something is a number
console.log(isNaN(NaN));        // true
console.log(isNaN(123));        // false
console.log(isFinite(Infinity)); // false
console.log(isFinite(123));      // true

4.3 The String Type

// Three ways to create strings
let singleQuotes = 'Hello with single quotes';
let doubleQuotes = "Hello with double quotes";
let backticks = `Hello with backticks (template literals)`;

// When to use which
let sentence1 = "It's a beautiful day";  // Double quotes allow apostrophe
let sentence2 = 'He said "Hello"';        // Single quotes allow double quotes
let sentence3 = `It's "JavaScript"`;      // Backticks allow both

// Escape characters
let escaped = "She said, \"Hello!\"";
let newLine = "First line\nSecond line";
let tab = "Column1\tColumn2";
let backslash = "This is a backslash: \\";

// String concatenation
let firstName = "John";
let lastName = "Doe";
let fullName = firstName + " " + lastName;  // "John Doe"
let greeting = "Hello, " + firstName + "!"; // "Hello, John!"

// Template literals (ES6+) - BEST WAY
let age = 30;
let templateString = `Hello, my name is ${firstName} ${lastName} and I am ${age} years old.`;
console.log(templateString);
// "Hello, my name is John Doe and I am 30 years old."

// Multi-line strings with template literals
let multiLine = `
This is line 1
This is line 2
This is line 3
`;
console.log(multiLine);

// String properties and methods
let text = "JavaScript";

console.log(text.length);           // 10 (number of characters)
console.log(text[0]);                // "J" (first character)
console.log(text[text.length - 1]);  // "t" (last character)
console.log(text.toUpperCase());     // "JAVASCRIPT"
console.log(text.toLowerCase());     // "javascript"
console.log(text.indexOf("Script")); // 4 (position where "Script" starts)
console.log(text.includes("Java"));  // true
console.log(text.slice(0, 4));       // "Java" (extract characters 0-3)
console.log(text.substring(4, 10));  // "Script"
console.log(text.replace("Java", "Type")); // "TypeScript"

// String comparison
console.log("a" < "b");              // true (lexicographic)
console.log("apple" < "banana");     // true
console.log("2" > "12");             // true (compares character by character: '2' > '1')

// Converting to string
let num = 123;
let str1 = String(num);              // "123"
let str2 = num.toString();           // "123"
let str3 = "" + num;                 // "123" (implicit conversion)

4.4 The Boolean Type

// Boolean values
let isActive = true;
let isLoggedIn = false;

// Boolean from comparisons
let isGreater = 5 > 3;     // true
let isLess = 5 < 3;         // false
let isEqual = 5 === 5;      // true
let isNotEqual = 5 !== 5;   // false

// Boolean from logical operations
let andOperation = true && true;    // true
let orOperation = true || false;    // true
let notOperation = !true;           // false

// Truthy and Falsy values
// FALSY values (evaluate to false):
console.log(Boolean(false));      // false
console.log(Boolean(0));          // false
console.log(Boolean(-0));         // false
console.log(Boolean(0n));         // false (BigInt zero)
console.log(Boolean(""));         // false (empty string)
console.log(Boolean(null));       // false
console.log(Boolean(undefined));  // false
console.log(Boolean(NaN));        // false

// TRUTHY values (everything else):
console.log(Boolean(true));       // true
console.log(Boolean(1));          // true
console.log(Boolean(-1));         // true
console.log(Boolean("hello"));    // true
console.log(Boolean("0"));        // true (non-empty string)
console.log(Boolean([]));         // true (empty array)
console.log(Boolean({}));         // true (empty object)
console.log(Boolean(function(){})); // true

// Practical use of truthy/falsy
let username = "";

if (username) {
    console.log("Welcome, " + username);
} else {
    console.log("Please enter a username"); // This runs
}

// Double negation to convert to boolean
console.log(!!"hello");  // true
console.log(!!0);         // false

// Variable declared but not initialized

// Function with no return value

// undefined vs undeclared
// console.log(notDeclared); // ReferenceError: notDeclared is not defined

4.6 The Null Type

// Explicitly setting to null
let user = null;
console.log(user); // null

// Common use cases
let selectedItem = null;       // No item selected

let cachedData = null;         // Cache is empty

let data = null;

    console.log("Data is null");
}

// null vs undefined
let a;              // a is undefined

console.log(a);     // undefined
console.log(b);     // null
console.log(typeof a); // "undefined"
console.log(typeof b); // "object" (historical bug in JavaScript)

4.7 Type Conversion and Coercion

Explicit Conversion (Type Casting)

// Convert to String
let num = 123;
let str1 = String(num);      // "123"
let str2 = num.toString();   // "123"
let str3 = (123).toString(); // "123"

// Convert to Number
let str = "123";
let num1 = Number(str);      // 123
let num2 = parseInt(str);    // 123
let num3 = parseFloat("3.14"); // 3.14
let num4 = +"456";           // 456 (unary plus operator)

// Convert to Boolean
let value = "hello";
let bool1 = Boolean(value);  // true
let bool2 = !!value;         // true (double negation)

// Special conversions
console.log(Number(""));           // 0
console.log(Number("  "));         // 0
console.log(Number("123abc"));     // NaN
console.log(Number(true));         // 1
console.log(Number(false));        // 0
console.log(Number(null));         // 0
console.log(Number(undefined));    // NaN

Implicit Conversion (Type Coercion)

// String coercion with +
console.log("5" + 3);        // "53" (number converted to string)
console.log("5" + "3");      // "53"
console.log("Hello" + 123);  // "Hello123"

// Numeric coercion with other operators
console.log("10" - 5);       // 5 (string converted to number)
console.log("10" * "2");     // 20 (both converted to numbers)
console.log("10" / "2");     // 5
console.log("10" - "5");     // 5
console.log("10" - "abc");   // NaN (can't convert "abc")

// Boolean coercion in conditions
if ("hello") {               // "hello" is truthy
    console.log("This runs");
}

if (0) {                     // 0 is falsy
    console.log("This doesn't run");
}

// Comparison coercion
console.log(5 == "5");       // true (loose equality - converts types)
console.log(5 === "5");      // false (strict equality - no conversion)
console.log(0 == false);     // true
console.log(0 === false);    // false
console.log(null == undefined); // true
console.log(null === undefined); // false

4.8 Checking Data Types

// Using typeof operator
console.log(typeof 42);              // "number"
console.log(typeof "hello");         // "string"
console.log(typeof true);            // "boolean"
console.log(typeof undefined);       // "undefined"
console.log(typeof null);            // "object" (historical bug)
console.log(typeof {name: "John"});  // "object"
console.log(typeof [1,2,3]);         // "object" (arrays are objects)
console.log(typeof function(){});    // "function"

// Better way to check for null
let value = null;
console.log(value === null);         // true

// Check for array
let arr = [1, 2, 3];
console.log(Array.isArray(arr));     // true
console.log(arr instanceof Array);    // true

// Check for NaN (NaN is the only value not equal to itself)
console.log(NaN === NaN);             // false
console.log(isNaN(NaN));              // true
console.log(isNaN("hello"));          // true (converts to number first)
console.log(Number.isNaN("hello"));   // false (doesn't convert)

// Using Object.prototype.toString (most accurate)
console.log(Object.prototype.toString.call(42));        // "[object Number]"
console.log(Object.prototype.toString.call("hello"));   // "[object String]"
console.log(Object.prototype.toString.call(null));      // "[object Null]"
console.log(Object.prototype.toString.call(undefined)); // "[object Undefined]"
console.log(Object.prototype.toString.call([]));        // "[object Array]"
console.log(Object.prototype.toString.call({}));        // "[object Object]"

Example 2: Shopping Cart

// Shopping cart variables
let cartItems = [];              // array (will store objects)
const TAX_RATE = 0.08;           // number (constant)
let subtotal = 0;                 // number
let discount = null;               // null (no discount initially)
let appliedCoupon = undefined;     // undefined

// Adding an item to cart
function addToCart(itemName, price, quantity = 1) {
    let item = {
        name: itemName,
        price: price,
        quantity: quantity
    };
    
    cartItems.push(item);
    subtotal += price * quantity;
    
    console.log(`Added ${quantity} x ${itemName} to cart`);
}

// Apply discount coupon
function applyCoupon(couponCode) {
    if (couponCode === "SAVE10") {
        discount = subtotal * 0.1;
        appliedCoupon = couponCode;
        console.log("Coupon applied: 10% off");
    } else {
        console.log("Invalid coupon code");
    }
}

// Calculate total
function calculateTotal() {
    let total = subtotal;
    
    if (discount !== null) {
        total -= discount;
    }
    
    let tax = total * TAX_RATE;
    total += tax;
    
    return {
        subtotal: subtotal,
        discount: discount,
        tax: tax,
        total: total
    };
}

addToCart("Laptop", 999.99);
addToCart("Mouse", 29.99, 2);

console.log("Tax: $" + totals.tax);

// Game variables
let playerName = "Player1";          // string

// Score functions
function increaseScore(points) {
    if (!isGameActive) {
        console.log("Game is not active");
        return;
    }
    
    score += points;
    console.log(`Score: ${score}`);
    
    // Check for high score
    if (score > highScore) {
        highScore = score;
        console.log("New high score!");
    }
}

    powerUp = type;
    lastPowerUpTime = Date.now();
    console.log(`Power-up activated: ${type}`);
}

function updateGame() {
    // Check if power-up expired (after 5 seconds)
    if (powerUp !== null && lastPowerUpTime) {
        let timeActive = Date.now() - lastPowerUpTime;
        if (timeActive > 5000) {
            powerUp = null;
            console.log("Power-up expired");
        }
    }
}

activatePowerUp("Speed Boost");
increaseScore(50);
// After 5 seconds...
updateGame(); // Power-up will be null

5.2 Best Practices and Common Pitfalls

DO's ✅

// 1. Use const by default, let when needed
const API_URL = "https://api.example.com";  // won't change
let currentUser = null;                       // will change

// 2. Use descriptive names
let numberOfActiveUsers = 15;   // Good
let nau = 15;                   // Bad

// 3. Initialize variables when declaring
let count = 0;                   // Good
let name = "";                   // Good
let items = [];                  // Good
let config = null;                // Good

// 4. Use template literals for strings with variables
let user = "John";
let greeting = `Hello, ${user}!`;  // Good
// let oldGreeting = "Hello, " + user + "!"; // Old way

// 5. Use strict equality (===) instead of loose (==)
if (count === 0) {  // Good
    // ...
}

// 6. Check for null/undefined properly
if (user === null || user === undefined) {
    // Handle case
}

// Modern way (optional chaining)
// if (user?.name) { ... }

DON'TS ❌

// 1. Don't use var
var oldWay = "Avoid this";  // Bad

// 2. Don't use implicit globals
function badFunction() {
    accidentalGlobal = "Oops!";  // Missing let/const - becomes global!
}

// 3. Don't use reserved words
// let class = "math";  // SyntaxError

// 4. Don't use confusing names
let data = "something";  // Too vague
let d = "something";     // Even worse

// 5. Don't rely on automatic semicolon insertion (use semicolons)
let a = 5
let b = 10  // Works but risky

// 6. Don't compare with loose equality
console.log(0 == false);    // true (but confusing)
console.log("" == false);    // true (but confusing)
console.log([] == false);    // true (very confusing)

// 7. Don't forget to declare variables in loops
for (i = 0; i < 5; i++) {  // Bad - i becomes global
    // ...
}

// 8. Don't reassign constants
const PI = 3.14;
// PI = 3.14159;  // TypeError
