1. INTRODUCTION TO CONTROL FLOW

1.1 What is Control Flow?

Imagine you're following a recipe. The instructions might say:

"If the dough is sticky, add more flour. Otherwise, knead for 10 minutes."

This is exactly how control flow works in programming. It's the order in which individual statements, instructions, or function calls are executed or evaluated. Without control flow, programs would simply execute line by line from top to bottom, unable to respond to different situations or user inputs.

Control flow is the fundamental mechanism that allows our programs to make decisions, repeat actions, and respond dynamically to different inputs and situations.

1.2 The Importance of Decision-Making in Code

Consider these everyday applications that rely on conditional logic:

  • Login Systems: "If the password is correct, grant access. Else, show an error message."
  • E-commerce Sites: "If the user is a premium member, apply a 20% discount. Else, show standard prices."
  • Games: "If the player's health reaches zero, end the game. Else, continue playing."
  • Forms: "If the email field is empty, display 'Required Field'. Else, proceed."
  • Navigation Apps: "If there's traffic on the current route, suggest an alternative."

Without conditionals, our programs would be like a elevator that only goes to one floor, regardless of which button you press!

1.3 Boolean Logic and Truthy/Falsy Values

Before diving into conditionals, we must understand what JavaScript evaluates as true or false.

Boolean Primitives

let isLoggedIn = true;
let isDeleted = false;

Truthy Values

In JavaScript, a truthy value is a value that is considered true when encountered in a Boolean context. All values are truthy unless they are defined as falsy.

// All of these evaluate to TRUE in a conditional
if (true)
if ({})           // empty object
if ([])           // empty array
if (42)           // any non-zero number
if ("0")          // any non-empty string (even "false" as a string!)
if ("hello")      // non-empty string
if (new Date())   // any object
if (Infinity)     // Infinity
if (-42)          // non-zero negative number

Falsy Values

A falsy value is a value that is considered false when encountered in a Boolean context. There are only 8 falsy values in JavaScript:

// The 8 FALSY values in JavaScript:
if (false)        // boolean false
if (null)         // null
if (undefined)    // undefined
if (0)            // number zero
if (-0)           // negative zero
if (0n)           // BigInt zero
if (NaN)          // Not a Number
if ("")           // empty string

// All of the above evaluate to FALSE

Understanding Truthy/Falsy in Practice

// This is why we can write concise checks:
let username = "";

if (username) {
    console.log("Welcome, " + username);
} else {
    console.log("Please enter a username"); // This runs because "" is falsy
}

// But be careful with "0" and "false" as strings:
let value = "false";  // This is a NON-empty string
if (value) {
    console.log("This will run!"); // "false" as a string is truthy!
}

let number = "0";     // This is a NON-empty string
if (number) {
    console.log("This will also run!"); // "0" as a string is truthy!
}

The Boolean() Function

console.log(Boolean("hello"));      // true
console.log(Boolean(""));           // false
console.log(Boolean(42));           // true
console.log(Boolean(0));            // false
console.log(Boolean([]));           // true (empty array)
console.log(Boolean({}));           // true (empty object)
console.log(Boolean(null));         // false
console.log(Boolean(undefined));    // false

The Double NOT Operator (!!)

console.log(!!"hello");     // true
console.log(!!"");          // false
console.log(!!42);          // true
console.log(!!0);           // false
console.log(!![]);          // true
console.log(!!null);        // false

2.5 Important Notes and Gotchas

Curly Braces Are Optional (But Don't Skip Them!)

// This works, but it's risky:
if (5 > 3)
    console.log("Five is greater than three"); // This runs
    console.log("This line runs ALWAYS!");     // NOT part of the if!

// Always use curly braces for clarity and safety:
if (5 > 3) {
    console.log("Five is greater than three");
    console.log("This is clearly part of the if block");
}

Assignment vs. Comparison

let x = 5;

// Common mistake: using = instead of ===
if (x = 10) {  // This assigns 10 to x, then checks if 10 is truthy (it is!)
    console.log("This will ALWAYS run!");
}

// Correct way:
if (x === 10) {
    console.log("This runs only if x equals 10");
}

Complex Conditions

let age = 25;
let hasLicense = true;
let isSuspended = false;

// You can combine multiple conditions
if (age >= 18 && hasLicense && !isSuspended) {
    console.log("You can legally drive!");
}

3.5 The Dangling Else Problem

// This is clear and works as expected:

    if (condition2)
        statement1;
    else
        statement2;  // This else belongs to if(condition2)

// Using curly braces makes it explicit:
if (condition1) {
    if (condition2) {
        statement1;
    } else {
        statement2;  // Clearly belongs to if(condition2)
    }
}

3.6 Best Practices for Using else

Be Explicit with Conditions

// Good: Clear and readable
if (userScore >= 100) {
    console.log("You reached level 2!");
} else {
    console.log(`You need ${100 - userScore} more points for next level.`);
}

// Avoid: Redundant condition
if (userScore >= 100) {
    console.log("You reached level 2!");
} 
if (userScore < 100) {  // This is redundant; use else instead
    console.log(`You need ${100 - userScore} more points.`);
}

Handle All Possible Cases

function getGradeDescription(grade) {
    if (grade >= 90) {
        return "Excellent";
    } else {
        return "Keep studying"; // Catches all grades below 90
    }
}

Consider the "Early Return" Pattern

// Without early return (using else)
function processOrder(order) {
    if (order.isValid) {
        // 20 lines of order processing
        return "Order processed";
    } else {
        return "Invalid order";
    }
}

// With early return (cleaner, no else needed)
function processOrder(order) {
    if (!order.isValid) {
        return "Invalid order";
    }
    
    // 20 lines of order processing
    // No need for else, as we've already returned
    return "Order processed";
}

4. THE else if LADDER

4.1 Handling Multiple Conditions

// Basic syntax
if (condition1) {
    // Code executes if condition1 is true
} else if (condition2) {
    // Code executes if condition1 is false AND condition2 is true
} else if (condition3) {
    // Code executes if condition1 and condition2 are false AND condition3 is true
} else {
    // Code executes if all above conditions are false
}

4.2 How It Works

let score = 85;

if (score >= 90) {
    console.log("Grade: A");
} else if (score >= 80) {
    console.log("Grade: B"); // This runs (85 >= 80)
} else if (score >= 70) {
    console.log("Grade: C"); // Skipped
} else if (score >= 60) {
    console.log("Grade: D"); // Skipped
} else {
    console.log("Grade: F"); // Skipped
}

Important: The conditions are checked in order, and execution stops at the first true condition.

    console.log("Senior"); // This runs for 65+

    console.log("Adult");  // Runs for 18-64
} else if (age >= 13) {
    console.log("Teenager"); // Runs for 13-17

    console.log("Child"); // Runs for 0-12

// Output: "Senior"

    console.log("Positive"); // This runs for ALL positive numbers

    console.log("Greater than 100"); // Never runs!
} else if (number > 1000) {
    console.log("Greater than 1000"); // Never runs!

// CORRECT - Specific to general

    console.log("Greater than 1000");

4.6 Best Practices for else if Ladders

Keep Conditions Mutually Exclusive When Possible

// Good: Conditions don't overlap
if (category === "A") {
    handleCategoryA();
} else if (category === "B") {
    handleCategoryB();
} else if (category === "C") {
    handleCategoryC();
} else {
    handleUnknownCategory();
}

Consider Using Switch for Many Related Conditions

// Many else ifs (okay but verbose)
if (day === "Monday") {
    console.log("Start of work week");
} else if (day === "Friday") {
    console.log("TGIF!");
} else if (day === "Saturday" || day === "Sunday") {
    console.log("Weekend!");
} else {
    console.log("Midweek");
}

// Switch statement (cleaner for this case)
switch(day) {
    case "Monday":
        console.log("Start of work week");
        break;
    case "Friday":
        console.log("TGIF!");
        break;
    case "Saturday":
    case "Sunday":
        console.log("Weekend!");
        break;
    default:
        console.log("Midweek");
}

Don't Nest Too Deeply

// Hard to read:
if (condition1) {
    if (condition2) {
        if (condition3) {
            // Do something
        } else {
            // Handle condition3 false
        }
    } else {
        // Handle condition2 false
    }
} else {
    // Handle condition1 false
}

// Better: Use logical operators
if (condition1 && condition2 && condition3) {
    // Do something
} else if (condition1 && condition2) {
    // Handle condition3 false
} else if (condition1) {
    // Handle condition2 false
} else {
    // Handle condition1 false
}

5.4 Alternatives to Deep Nesting

Technique 1: Early Returns (Guard Clauses)

// Much cleaner with guard clauses:
function processUserData(user) {
    if (!user) {
        console.log("No user provided");
        return;
    }
    
    if (!user.isActive) {
        console.log("User account is inactive");
        return;
    }
    
    if (!user.hasCompletedProfile) {
        console.log("Profile incomplete");
        return;
    }
    
    if (!user.emailVerified) {
        console.log("Email not verified");
        return;
    }
    
    if (!user.subscriptionActive) {
        console.log("Subscription not active");
        return;
    }
    
    // 20 lines of code here
    console.log("User can access premium content");
}

Technique 2: Combine Conditions with Logical Operators

// Instead of deep nesting:
if (user) {
    if (user.age >= 18) {
        if (user.hasID) {
            if (user.notBlacklisted) {
                console.log("Can enter the venue");
            }
        }
    }
}

// Combine with &&:
if (user && user.age >= 18 && user.hasID && user.notBlacklisted) {
    console.log("Can enter the venue");
}

if (isEligibleForVenue(user)) {

8. COMMON PITFALLS AND BEST PRACTICES

8.1 Common Mistakes to Avoid

Mistake 1: Using Assignment Instead of Comparison

// WRONG
if (username = "admin") { // Always true (assignment)
    console.log("Welcome admin");
}

// CORRECT
if (username === "admin") {
    console.log("Welcome admin");
}

Mistake 2: Forgetting About Type Coercion

// WRONG - Loose equality can be confusing
if (1 == "1") { // true, but may not be what you want
    console.log("They're equal");
}

// CORRECT - Use strict equality
if (1 === "1") { // false
    console.log("They're not equal");
}

// Common issue:
if (0 == false) { // true (coerced)
    console.log("This runs!");
}

if (0 === false) { // false (different types)
    console.log("This doesn't run!");
}

Mistake 3: Overly Complex Conditions

// WRONG - Too complex

    // handle
}

// CORRECT - Break it down
const isActiveUser = user && user.isActive && !user.isSuspended;
const hasWriteAccess = ["admin", "editor"].includes(user?.role) || user?.permissions?.includes("write");
const isRecentUser = user?.lastLogin > oneMonthAgo || user?.hasActiveSubscription;

if (isActiveUser && hasWriteAccess && isRecentUser) {
    // handle
}

Mistake 4: Not Handling Edge Cases

// WRONG - Crashes if users is null
function getFirstUserName(users) {
    return users[0].name;
}

// CORRECT - Handle edge cases
function getFirstUserName(users) {
    if (!users || users.length === 0 || !users[0]) {
        return "No users";
    }
    return users[0].name;
}

Mistake 5: Deep Nesting

// WRONG - Pyramid of doom
if (condition1) {
    if (condition2) {
        if (condition3) {
            if (condition4) {
                // do something
            }
        }
    }
}

// CORRECT - Use guard clauses or combine conditions
if (!condition1 || !condition2 || !condition3 || !condition4) {
    return;
}
// do something

function validateLogin(username, password, rememberMe) {
    // Check username
    if (!username || username.length < 3) {
        return {
            success: false,
            message: "Username must be at least 3 characters",
            sessionDays: 0
        };
    }
    
    // Check password length
    if (!password || password.length < 8) {
        return {
            success: false,
            message: "Password must be at least 8 characters",
            sessionDays: 0
        };
    }
    
    // Check for number and uppercase in password
    let hasNumber = false;
    let hasUppercase = false;
    
    for (let i = 0; i < password.length; i++) {
        let char = password[i];
        if (char >= '0' && char <= '9') {
            hasNumber = true;
        }
        if (char >= 'A' && char <= 'Z') {
            hasUppercase = true;
        }
    }
    
    if (!hasNumber || !hasUppercase) {
        return {
            success: false,
            message: "Password must contain at least one number and one uppercase letter",
            sessionDays: 0
        };
    }
    
    // All validations passed
    return {
        success: true,
        message: "Login successful",
        sessionDays: rememberMe ? 30 : 1
    };
}

    // runs if all conditions are false
}

Ternary Operator

let result = condition ? valueIfTrue : valueIfFalse;

Truthy and Falsy Values

// Falsy (evaluate to false)
false, 0, -0, 0n, "", null, undefined, NaN

// Everything else is truthy

Comparison Operators

==  // loose equality (with type coercion)
=== // strict equality (no type coercion)
!=  // loose inequality
!== // strict inequality
>   // greater than
<   // less than
>=  // greater than or equal
<=  // less than or equal

Logical Operators

&&  // AND - all conditions must be true
||  // OR - at least one condition must be true
!   // NOT - reverses boolean value

Modern Operators (ES2020+)

??  // nullish coalescing (checks only null/undefined)
?.  // optional chaining (safe property access)

Key Takeaways

  1. Control flow is how programs make decisions
  2. If statements are the foundation of conditional logic
  3. Else provides an alternative path
  4. Else if handles multiple conditions
  5. Ternary operators offer concise syntax for simple conditions
  6. Truthy/falsy understanding is crucial for effective conditionals
  7. Readability should always be prioritized over cleverness
  8. Edge cases must be handled to prevent bugs
  9. Guard clauses and early returns reduce nesting
  10. Practice is essential for mastering conditionals

Common Patterns Cheat Sheet

Default Values

let name = inputName || "Guest";          // old way
let name = inputName ?? "Guest";           // new way (nullish)

Guard Clauses

function process(data) {
    if (!data) return;
    if (!data.isValid) return;
    // main logic here
}

Safe Property Access

let city = user?.address?.city ?? "Unknown";

let element = condition ?  : ;

Multiple Conditions

let isValid = age >= 18 && hasLicense && !isSuspended;
