1. INTRODUCTION TO WEB TYPOGRAPHY

1.1 What is Typography?

Typography is the art and technique of arranging type to make written language legible, readable, and appealing when displayed. In web design, typography plays a crucial role in:

  • User Experience (UX): Good typography makes content easier to read and navigate
  • Brand Identity: Font choices convey personality and tone
  • Visual Hierarchy: Different text styles guide users through content
  • Accessibility: Proper typography ensures content is accessible to all users

2. THE font-family PROPERTY

2.1 Definition and Purpose

The font-family property specifies the typeface or font for an element. It can hold several font names as a "fallback" system, where the browser tries each font until it finds one available on the user's system.

2.2 Syntax and Basic Usage

/* Basic syntax */
selector {
    font-family: "Font Name", generic-family, fallback;
}

/* Examples */
body {
    font-family: Arial, Helvetica, sans-serif;
}

h1 {
    font-family: "Times New Roman", Times, serif;
}

.special-text {
    font-family: "Courier New", Courier, monospace;
}

2.3 Types of Font Families

2.3.1 Generic Font Families

CSS defines five generic font families that serve as absolute fallbacks:

Generic Family Description Examples
serif Fonts with small lines or strokes at the ends of characters Times New Roman, Georgia, Garamond
sans-serif Fonts without serifs (clean, modern look) Arial, Helvetica, Verdana
monospace All characters have the same width Courier New, Consolas, Monaco
cursive Fonts that mimic handwriting Comic Sans MS, Brush Script
fantasy Decorative fonts (rarely used for body text) Impact, Papyrus

2.3.2 Web-Safe Fonts

Web-safe fonts are fonts that come pre-installed on most operating systems:

2.3.3 Web Fonts (Custom Fonts)

Using @font-face or services like Google Fonts to use custom fonts:

/* Using Google Fonts (in HTML) */
<link href="https://fonts.googleapis.com/css2?family=Open+Sans&display=swap" rel="stylesheet">

/* In CSS */
body {
    font-family: 'Open Sans', sans-serif;
}

/* Using @font-face */
@font-face {
    font-family: 'MyCustomFont';
    src: url('fonts/myfont.woff2') format('woff2'),
         url('fonts/myfont.woff') format('woff');
    font-weight: normal;
    font-style: normal;
}

body {
    font-family: 'MyCustomFont', Arial, sans-serif;
}

2.4 Font Family Best Practices

2.4.1 Quoting Font Names

/* When to use quotes */
font-family: "Times New Roman", serif;  /* Multiple words - quote */
font-family: 'Helvetica Neue', sans-serif; /* Multiple words - quote */
font-family: Arial, sans-serif;         /* Single word - no quote needed */
font-family: "Arial", sans-serif;        /* Also valid, but unnecessary */

2.4.2 The Fallback Chain

Always end with a generic family:

/* Good */
font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;

/* Poor (no generic fallback) */
font-family: 'Segoe UI', Tahoma, Geneva, Verdana;

2.4.3 System Font Stacks

Modern approach using native system fonts:

/* System font stack */
body {
    font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 
                 Roboto, Oxygen, Ubuntu, Cantarell, 
                 'Open Sans', 'Helvetica Neue', sans-serif;
}

3. THE font-size PROPERTY

3.1 Definition and Purpose

The font-size property sets the size of the text. It's one of the most important properties for readability and visual hierarchy.

3.2 Absolute Units

3.2.1 Pixels (px)

The most common unit, providing precise control:

p {
    font-size: 16px;  /* Standard paragraph size */
}

h1 {
    font-size: 32px;  /* Heading size */
}

small {
    font-size: 12px;  /* Small text */
}

Pros and Cons of Pixels:
- ✅ Precise control
- ✅ Predictable across devices
- ❌ Not scalable for users with visual impairments (if user changes browser's default font size)
- ❌ Less accessible for responsive design

3.2.2 Points (pt)

Used primarily for print stylesheets:

@media print {
    body {
        font-size: 12pt;  /* Standard print size */
    }
}

3.3 Relative Units

3.3.1 em Units

Relative to the parent element's font size:

body {
    font-size: 16px;  /* Base size */
}

.parent {
    font-size: 20px;
}

.child {
    font-size: 0.8em;  /* 16px (20px × 0.8) */
}

.grandchild {
    font-size: 1.2em;  /* 19.2px (16px × 1.2) */
}

The "em Compounding" Problem:

<div style="font-size: 1.2em;">  <!-- 19.2px -->
    <div style="font-size: 1.2em;">  <!-- 23.04px -->
        <div style="font-size: 1.2em;">  <!-- 27.648px -->
            Text gets progressively larger
        </div>
    </div>
</div>

3.3.2 rem Units (Root em)

Relative to the root (html) element's font size, avoiding compounding issues:

html {
    font-size: 16px;  /* Default browser size */
}

body {
    font-size: 1rem;  /* 16px */
}

h1 {
    font-size: 2rem;  /* 32px */
}

.nested {
    font-size: 1.5rem;  /* 24px - unaffected by nesting */
}

rem is now the preferred unit for most web projects:

html {
    font-size: 16px;  /* Base */
}

/* Responsive scaling */
@media (max-width: 768px) {
    html {
        font-size: 14px;  /* All rem units scale automatically */
    }
}

h1 { font-size: 2.5rem; }  /* 40px on desktop, 35px on mobile */
h2 { font-size: 2rem; }    /* 32px on desktop, 28px on mobile */
p { font-size: 1rem; }     /* 16px on desktop, 14px on mobile */

3.3.3 Percentage (%)

Relative to the parent element's size:

body {
    font-size: 16px;
}

p {
    font-size: 100%;  /* 16px */
}

h1 {
    font-size: 200%;  /* 32px */
}

.small {
    font-size: 75%;   /* 12px (if parent is 16px) */
}

3.3.4 Viewport Units

Relative to the viewport size (useful for responsive typography):

h1 {
    font-size: 5vw;  /* 5% of viewport width */
}

.hero-text {
    font-size: 10vmin;  /* 10% of the smaller viewport dimension */
}

.banner {
    font-size: 4vmax;   /* 4% of the larger viewport dimension */
}

Caution with viewport units:

/* Problem: Text can become too small on mobile */
h1 {
    font-size: 5vw;  /* On iPhone SE (375px): 18.75px - too small */
}

/* Solution: Use clamp() for responsive scaling */
h1 {
    font-size: clamp(2rem, 5vw, 4rem);  /* Min, preferred, max */
}

3.4 Keyword Values

3.4.1 Absolute Size Keywords

p {
    font-size: medium;  /* Default (usually 16px) */
}

.small-text {
    font-size: small;
}

.large-text {
    font-size: large;
}

/* Available keywords: xx-small, x-small, small, medium, 
   large, x-large, xx-large, xxx-large */

3.4.2 Relative Size Keywords

p {
    font-size: larger;   /* One step larger than parent */
    font-size: smaller;  /* One step smaller than parent */
}

3.5 Best Practices for Font Sizing

3.5.1 Recommended Font Sizes

Element Recommended Size (rem) Notes
Body text 1rem (16px) Base for readability
Small text 0.875rem (14px) Captions, footnotes
h1 2.5rem (40px) Main heading
h2 2rem (32px) Section heading
h3 1.75rem (28px) Sub-section
h4 1.5rem (24px) Group heading
h5 1.25rem (20px) Minor heading
h6 1rem (16px) Smallest heading
Navigation 1rem (16px) Easy to click
Buttons 1rem (16px) Readable call-to-action

3.5.2 Responsive Typography Pattern

/* Modern responsive typography setup */
html {
    font-size: 16px;
}

@media (min-width: 768px) {
    html {
        font-size: 17px;  /* Slightly larger on tablets */
    }
}

@media (min-width: 1200px) {
    html {
        font-size: 18px;  /* Larger on desktops */
    }
}

/* Or using viewport units with fallback */
h1 {
    font-size: 2rem;  /* Fallback */
    font-size: clamp(2rem, 5vw, 4rem);  /* Modern browsers */
}

3.5.3 The 62.5% Technique

A popular technique for easier rem calculations:

html {
    font-size: 62.5%;  /* 16px × 0.625 = 10px */
}

/* Now 1rem = 10px */
body {
    font-size: 1.6rem;  /* 16px */
}

h1 {
    font-size: 3.2rem;  /* 32px */
}

p {
    font-size: 1.4rem;  /* 14px */
}

4. THE font-weight PROPERTY

4.1 Definition and Purpose

The font-weight property controls the thickness or boldness of the text. It helps create visual hierarchy and emphasis.

4.2 Keyword Values

.normal-text {
    font-weight: normal;  /* Standard weight (usually 400) */
}

.bold-text {
    font-weight: bold;    /* Bold weight (usually 700) */
}

.lighter-text {
    font-weight: lighter; /* Lighter than parent element */
}

.bolder-text {
    font-weight: bolder;  /* Bolder than parent element */
}

4.3 Numeric Values (100-900)

CSS allows precise control with numeric values from 100 to 900:

.thin {
    font-weight: 100;  /* Thin / Hairline */
}

.extra-light {
    font-weight: 200;  /* Extra Light / Ultra Light */
}

.light {
    font-weight: 300;  /* Light */
}

.regular {
    font-weight: 400;  /* Regular / Normal */
}

.medium {
    font-weight: 500;  /* Medium */
}

.semi-bold {
    font-weight: 600;  /* Semi Bold / Demi Bold */
}

.bold {
    font-weight: 700;  /* Bold */
}

.extra-bold {
    font-weight: 800;  /* Extra Bold / Ultra Bold */
}

.black {
    font-weight: 900;  /* Black / Heavy */
}

4.5 Font Weight Availability

Not all fonts have all weights available:

/* If a font doesn't have weight 300, it will use the 
   closest available weight */

h1 {
    font-family: 'SomeFont', sans-serif;
    font-weight: 300;  /* May show as 400 if 300 unavailable */
}

4.5.1 Checking Available Weights

/* With Google Fonts, specify which weights to load */
<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@300;400;700;900&display=swap" rel="stylesheet">

/* Then use them */
.light { font-weight: 300; }
.regular { font-weight: 400; }
.bold { font-weight: 700; }
.black { font-weight: 900; }

4.6 Relative Weights: lighter and bolder

.container {
    font-weight: 500;  /* Medium */
}

.container p {
    font-weight: lighter;  /* 400 if available, otherwise lighter than 500 */
}

.container strong {
    font-weight: bolder;   /* 700 if available, otherwise bolder than 500 */
}

4.7 Best Practices for Font Weight

4.7.1 Creating Typographic Hierarchy

/* Good hierarchy using font-weight */
h1 {
    font-weight: 700;  /* Bold */
}

h2 {
    font-weight: 600;  /* Semi-bold */
}

h3 {
    font-weight: 500;  /* Medium */
}

p {
    font-weight: 400;  /* Regular */
}

small, .caption {
    font-weight: 300;  /* Light */
}

strong, b {
    font-weight: 700;  /* Bold for emphasis */
}

4.7.2 Combining with Font Size

.news-article {
    font-size: 1.1rem;
    font-weight: 400;
}

.news-article .headline {
    font-size: 2.2rem;
    font-weight: 700;  /* Bold and large = very prominent */
}

.news-article .byline {
    font-size: 0.9rem;
    font-weight: 300;  /* Light and small = subtle */
}

.news-article .summary {
    font-size: 1rem;
    font-weight: 500;  /* Medium weight for emphasis */
}

4.7.3 Accessibility Considerations

/* Avoid using font-weight alone to convey meaning */
.error-message {
    font-weight: 700;  /* Also use color, icon, or other indicators */
    color: #d32f2f;
}

/* Ensure sufficient contrast with font weight */
.light-text {
    font-weight: 300;
    color: #333;  /* Darker color for light weight to maintain contrast */
}

5. THE font-style PROPERTY

5.1 Definition and Purpose

The font-style property specifies whether text should be displayed in a normal, italic, or oblique style. It's primarily used for emphasis and to differentiate certain types of content.

5.2 Available Values

.normal {
    font-style: normal;  /* Standard upright text */
}

.italic {
    font-style: italic;  /* Italic version of the font */
}

.oblique {
    font-style: oblique; /* Slanted version of the normal font */
}

.oblique-10deg {
    font-style: oblique 10deg; /* Oblique with specific angle */
}

5.3 Italic vs. Oblique: Understanding the Difference

┌────────────────────────────────────────────────────────────┐
│ ITALIC                        OBLIQUE                       │
├────────────────────────────────────────────────────────────┤
│ Designed separately           Algorithmically slanted       │
│ Has unique characteristics    Just skewed version           │
│ Usually more elegant          Can look distorted            │
│ Available as separate font    Generated from regular        │
└────────────────────────────────────────────────────────────┘

Visual Example:

Regular:    The quick brown fox
Italic:     *The quick brown fox* (specially designed)
Oblique:    \The quick brown fox\ (just tilted)

5.4 Practical Usage

5.4.1 Common Use Cases

/* Emphasis */
em {
    font-style: italic;
}

/* Quotes and citations */
cite {
    font-style: italic;
}

/* Foreign words */
.foreign-word {
    font-style: italic;
}

/* Book and movie titles */
.movie-title {
    font-style: italic;
}

/* Technical terms */
.latin-term {
    font-style: italic;
}

5.4.2 Complete Example

<article class="blog-post">
    <h1>The Art of <span class="foreign-word">Espresso</span></h1>
    <p class="intro">
        As <cite>James Hoffmann</cite> writes in 
        <span class="movie-title">The World Atlas of Coffee</span>:
    </p>
    <blockquote>
        <p>
            <em>The perfect espresso</em> requires attention to detail. 
            The term <span class="latin-term">créma</span> refers to 
            the golden foam on top.
        </p>
    </blockquote>
</article>

.blog-post {
    font-family: Georgia, serif;
    line-height: 1.6;
}

.foreign-word {
    font-style: italic;
    font-weight: 500;
}

cite {
    font-style: italic;
    font-weight: 300;
}

.movie-title {
    font-style: italic;
    text-decoration: underline;
}

em {
    font-style: italic;
    background-color: #fff3cd;
    padding: 0 3px;
}

.latin-term {
    font-style: italic;
    font-family: 'Times New Roman', serif;
}

5.5 Browser Support and Rendering

/* If italic isn't available */
p {
    font-style: italic;
    /* Browser will simulate oblique if no italic font exists */
}

/* Fallback behavior */
.fancy-text {
    font-family: 'FancyFont', 'Times New Roman', serif;
    font-style: italic;
    /* If FancyFont has italic, use it
       If not, use Times New Roman italic
       If Times has no italic, simulate oblique */
}

5.6 Best Practices

5.6.1 Semantic HTML vs. Visual Style

/* GOOD: Semantic with CSS */
em {
    font-style: italic;  /* em means emphasis */
}

/* BAD: Using italic for non-emphasized content */
.photo-credit {
    font-style: italic;  /* Better to use <cite> or specific class */
}

/* GOOD: Using specific class */
.photo-credit {
    font-style: italic;
}

5.6.2 Readability Considerations

/* Avoid long passages in italic */
.long-italic {
    font-style: italic;  /* Hard to read in large blocks */
}

/* Use italic sparingly */
.article p {
    font-style: normal;  /* Body text should be normal */
}

.article p.important-note {
    font-style: italic;  /* Short emphasis only */
}

5.6.3 Combining with Other Properties

.special-text {
    font-style: italic;
    font-weight: 600;  /* Semi-bold italic */
    font-family: 'Georgia', serif;  /* Serif fonts often have nice italics */
}

6. THE line-height PROPERTY

6.1 Definition and Purpose

The line-height property controls the amount of space between lines of text. It significantly affects readability and the overall visual texture of text blocks.

6.2 Understanding Line Height

┌─────────────────────────────────────────────┐
│                                             │
│  This is the first line of text.            │←───┐
│                                             │    │ line-height
│  This is the second line of text.           │←───┘
│                                             │
│  This is the third line of text.            │
│                                             │
└─────────────────────────────────────────────┘

Leading (pronounced "ledding") is the space between lines,
traditionally measured from baseline to baseline.

6.3 Line Height Values

6.3.1 Unitless Numbers (Recommended)

p {
    line-height: 1.5;  /* 1.5 × font-size */
}

/* Unitless is preferred because it scales with font-size */
body {
    font-size: 16px;
    line-height: 1.5;  /* 24px computed */
}

h1 {
    font-size: 32px;
    line-height: 1.2;  /* 38.4px computed - tighter for headings */
}

6.3.2 Length Values (px, em, rem)

p {
    line-height: 24px;   /* Fixed pixel height */
    line-height: 1.5em;  /* Relative to element's font-size */
    line-height: 1.5rem; /* Relative to root font-size */
}

Important Note on em and inheritance:

.container {
    font-size: 20px;
    line-height: 1.5em;  /* 30px */
}

.container p {
    font-size: 16px;
    /* line-height inherits 30px, not recalculated for 16px */
    /* This can cause spacing issues! */
}

/* Better */
.container {
    font-size: 20px;
    line-height: 1.5;  /* Unitless */
}

.container p {
    font-size: 16px;
    /* line-height recalculates: 16px × 1.5 = 24px */
}

6.3.3 Percentage Values

p {
    line-height: 150%;  /* 1.5 × font-size */
}

/* Percentage inherits the computed value, not the multiplier */
body {
    font-size: 16px;
    line-height: 150%;  /* 24px computed */
}

h2 {
    font-size: 24px;
    line-height: 150%;  /* Inherits 24px? No! 36px (24px × 1.5) */
}

6.3.4 Normal Keyword

p {
    line-height: normal;  /* Browser default (usually 1.2) */
}

/* normal is approximately 1.2, but varies by font and browser */

6.4 Recommended Line Heights

Text Type Recommended Line Height Notes
Body text 1.5 - 1.6 Optimal readability
Headings 1.1 - 1.3 Tighter for larger text
Small text 1.5 - 1.8 More space improves readability
Long articles 1.6 - 1.8 Extra space for long reads
Navigation 1 - 1.2 Usually single lines
Buttons 1 - 1.2 Vertical centering

6.5 Practical Examples

6.5.1 Basic Text Spacing

/* Readable article */
article {
    font-size: 18px;
    line-height: 1.6;  /* 28.8px - good for reading */
    max-width: 700px;
    margin: 0 auto;
}

/* Headings with appropriate spacing */
h1 {
    font-size: 48px;
    line-height: 1.2;  /* 57.6px */
    margin-bottom: 0.5em;
}

h2 {
    font-size: 36px;
    line-height: 1.3;  /* 46.8px */
}

/* Dense navigation */
.nav-menu {
    line-height: 1;  /* Exactly the font-size */
}

6.5.2 Vertical Centering with Line Height

/* Centering single lines of text */
.button {
    height: 50px;
    line-height: 50px;  /* Same as height */
    text-align: center;
    padding: 0 20px;
}

/* Multiple lines need different approach */
.notification {
    padding: 15px 20px;  /* Use padding instead */
    line-height: 1.5;
}

6.5.3 Creating Vertical Rhythm

/* Consistent vertical spacing throughout page */
:root {
    --base-font-size: 16px;
    --base-line-height: 1.5;
    --rhythm: calc(var(--base-font-size) * var(--base-line-height));
}

body {
    font-size: var(--base-font-size);
    line-height: var(--base-line-height);
}

h1 {
    font-size: 2.5rem;
    line-height: 1.2;
    margin-bottom: var(--rhythm);
}

p {
    margin-bottom: var(--rhythm);
}

.component {
    margin-bottom: calc(var(--rhythm) * 2);
}

6.6 Line Height and Accessibility

/* Ensure sufficient line spacing */
body {
    line-height: 1.5;  /* WCAG Success Criterion 1.4.8 */
}

/* For users who need more spacing */
body.spaced {
    line-height: 2;
}

/* Avoid too tight spacing */
.tight-text {
    line-height: 1.2;  /* Only for short text, not paragraphs */
}

6.7 Troubleshooting Line Height Issues

6.7.1 Descenders Collision

/* Problem: Descenders of g, j, p, q, y touching next line */
p {
    line-height: 1.2;  /* Too tight for body text */
    font-size: 24px;   /* Large text makes it worse */
}

/* Solution */
p {
    line-height: 1.5;  /* Give descenders room */
}

6.7.2 Inline Elements and Line Height

/* Line height affects inline elements differently */
p {
    line-height: 2;
}

span.highlight {
    background-color: yellow;
    /* The span inherits line-height, but the background
       only covers the content area, not the line height */
}

7. THE text-align PROPERTY

7.1 Definition and Purpose

The text-align property controls the horizontal alignment of text within its containing element. It's fundamental for creating visual structure and readability.

7.2 Available Values

.left-text {
    text-align: left;      /* Default in LTR languages */
}

.right-text {
    text-align: right;     /* Default in RTL languages */
}

.center-text {
    text-align: center;    /* Centered alignment */
}

.justified-text {
    text-align: justify;   /* Spreads text to fill the line */
}

.start-text {
    text-align: start;     /* left in LTR, right in RTL */
}

.end-text {
    text-align: end;       /* right in LTR, left in RTL */
}

8. THE text-decoration PROPERTY

8.1 Definition and Purpose

The text-decoration property adds decorative lines to text. It's commonly used for links, but has many other applications for visual enhancement.

8.2 The Text Decoration Properties

8.2.1 Shorthand vs. Longhand

/* Shorthand (recommended for most cases) */
.underlined {
    text-decoration: underline wavy blue 2px;
}

/* Longhand properties */
.link {
    text-decoration-line: underline;
    text-decoration-style: solid;
    text-decoration-color: #007bff;
    text-decoration-thickness: 2px;
}

8.3 text-decoration-line

/* Basic line values */
.underline {
    text-decoration-line: underline;  /* Line below text */
}

.overline {
    text-decoration-line: overline;   /* Line above text */
}

.line-through {
    text-decoration-line: line-through;  /* Line through text */
}

/* Multiple lines */
.fancy {
    text-decoration-line: underline overline;  /* Both above and below */
}

.none {
    text-decoration-line: none;  /* Remove decoration (common on links) */
}

8.4 text-decoration-style

    text-decoration-line: underline;

    text-decoration-style: dashed;  /* Dashed line */
}

.double-underline {
    text-decoration-line: underline;
    text-decoration-style: double;  /* Double line */
}

12. COMMON PITFALLS AND TROUBLESHOOTING

12.1 Font Loading Issues

12.1.1 Flash of Unstyled Text (FOUT)

/* Problem: Text appears in fallback font, then switches */
@font-face {
    font-family: 'CustomFont';
    src: url('font.woff2');
    /* No font-display specified */
}

/* Solution 1: Use font-display */
@font-face {
    font-family: 'CustomFont';
    src: url('font.woff2');
    font-display: swap;  /* Show fallback, then swap */
}

/* Solution 2: Use font loading API (advanced) */
document.fonts.ready.then(function() {
    document.documentElement.classList.add('fonts-loaded');
});

12.1.2 Flash of Invisible Text (FOIT)

/* Problem: Text invisible while font loads */
@font-face {
    font-family: 'CustomFont';
    src: url('font.woff2');
    /* Browser may hide text up to 3 seconds */
}

/* Solution: Use font-display */
@font-face {
    font-family: 'CustomFont';
    src: url('font.woff2');
    font-display: fallback;  /* Short FOIT then swap */
}

12.2 Line Height Problems

12.2.1 Vertical Spacing Issues

/* Problem: Images breaking line height */
p {
    line-height: 1.5;
}

p img {
    height: 100px;  /* Increases line height unexpectedly */
}

/* Solution */
p img {
    height: 100px;
    display: block;  /* Remove from text flow */
    margin: 10px 0;
}

12.2.2 Button Text Alignment

/* Problem: Text not centered vertically */
.button {
    height: 50px;
    font-size: 16px;
    /* line-height not specified */
}

/* Solution */
.button {
    height: 50px;
    font-size: 16px;
    line-height: 50px;  /* Match height for single line */
    text-align: center;
    padding: 0 20px;
}

/* Better solution for multi-line */
.button {
    padding: 15px 20px;
    line-height: 1.2;
    display: inline-flex;
    align-items: center;
    justify-content: center;
}

12.3 Text Decoration Issues

12.3.1 Underline Too Close to Text

/* Problem: Underline touching descenders */
.link {
    text-decoration: underline;
    /* Underline touches 'g', 'j', 'p', 'q', 'y' */
}

/* Solution */
.link {
    text-decoration: underline;
    text-underline-offset: 3px;  /* Add space */
    text-decoration-thickness: 2px;
}

12.3.2 Multi-line Underline

/* Problem: Underline broken across lines */
.multi-line {
    text-decoration: underline;
    /* Underline may appear under each line separately */
}

/* Solution: Use border-bottom for multi-line */
.multi-line {
    text-decoration: none;
    border-bottom: 2px solid #000;
    padding-bottom: 2px;
    display: inline;  /* Border will span multiple lines */
}

12.4 Font Weight Issues

12.4.1 Weight Not Available

/* Problem: Weight doesn't exist in font */
p {
    font-family: 'Roboto', sans-serif;
    font-weight: 450;  /* Not a standard weight */
}

/* Solution: Use available weights */
p {
    font-family: 'Roboto', sans-serif;
    font-weight: 400;  /* Use standard weight */
}

/* Or load the specific weight */
@import url('https://fonts.googleapis.com/css2?family=Roboto:wght@400;500;700&display=swap');

12.4.2 Synthetic Bold

/* Problem: Browser synthesizes bold */
p {
    font-family: 'LightFont', sans-serif;
    font-weight: 700;  /* Font doesn't have bold - browser fakes it */
}

/* Solution: Use appropriate font */
p {
    font-family: 'LightFont', sans-serif;
    font-weight: 300;  /* Use available weight */
}

/* Or use a font family with bold version */
p {
    font-family: 'LightFont', 'BoldFont', sans-serif;
    font-weight: 700;
}

12.5 Responsive Typography Issues

12.5.1 Text Too Small on Mobile

/* Problem: Fixed sizes don't scale */
h1 {
    font-size: 48px;  /* Too large on mobile? */
}

/* Solution 1: Media queries */
h1 {
    font-size: 32px;
}

@media (min-width: 768px) {
    h1 {
        font-size: 48px;
    }
}

/* Solution 2: Viewport units with clamp */
h1 {
    font-size: clamp(2rem, 5vw, 3rem);
}

/* Solution 3: rem scaling */
html {
    font-size: 16px;
}

@media (max-width: 768px) {
    html {
        font-size: 14px;  /* All rem units scale */
    }
}

12.5.2 Text Overflow on Small Screens

/* Problem: Long words break layout */
.long-word {
    /* Text overflows container */
}

/* Solution */
.long-word {
    overflow-wrap: break-word;  /* Break long words */
    word-wrap: break-word;       /* Older browsers */
    word-break: break-word;      /* Additional support */
}

/* For URLs */
.url {
    word-break: break-all;  /* Break anywhere if needed */
}

14.3 Common Pitfalls to Avoid

  1. ❌ Using pixels exclusively → Use rem for better accessibility
  2. ❌ Forgetting fallback fonts → Always end with a generic family
  3. ❌ Line height too tight → Minimum 1.5 for body text
  4. ❌ Removing focus outlines → Replace with visible indicators
  5. ❌ Fixed font sizes → Make them responsive
  6. ❌ Overusing !important → Use specific selectors instead
  7. ❌ Ignoring vertical rhythm → Maintain consistent spacing
