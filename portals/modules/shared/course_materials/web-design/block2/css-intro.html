1. INTRODUCTION TO CSS

What is CSS?

CSS stands for Cascading Style Sheets. It is a stylesheet language used to describe the presentation of a document written in HTML or XML. CSS defines how elements should be rendered on screen, on paper, in speech, or on other media.

In simple terms:

  • HTML is the structure and content (the nouns)
  • CSS is the presentation and styling (the adjectives)
<!-- HTML provides structure -->
<p>This is a paragraph.</p>

<!-- CSS adds style -->
<style>
  p {
    color: blue;
    font-size: 16px;
    font-family: Arial, sans-serif;
  }
</style>

The Purpose of CSS

1. Visual Styling

CSS controls the visual appearance of web elements:

  • Colors: Text color, background color, border colors
  • Typography: Fonts, sizes, weights, line heights, spacing
  • Layout: Positioning, sizing, margins, padding
  • Effects: Shadows, gradients, animations, transitions

2. Separation of Content from Presentation

This is the most important purpose of CSS. By separating content (HTML) from presentation (CSS), we gain numerous benefits:

  • Maintainability: Change the entire look of a website by modifying one CSS file
  • Accessibility: Users can override styles for better readability
  • Device Compatibility: Same HTML can be styled differently for screen, print, or mobile
  • Faster Development: Consistent styling across multiple pages
  • Cleaner Code: HTML becomes cleaner and more readable

3. Consistency Across Pages

CSS ensures that styling remains consistent throughout an entire website:

/* One rule affects every h1 on every page */
h1 {
  color: #2c3e50;
  font-family: 'Helvetica', sans-serif;
  border-bottom: 2px solid #3498db;
}

4. Responsive Design

CSS enables websites to adapt to different screen sizes and devices:

/* Styles adapt based on screen width */
@media (max-width: 768px) {
  body {
    font-size: 14px;
  }
  
  .sidebar {
    display: none;
  }
}

5. User Experience Enhancements

CSS adds interactivity and visual feedback:

/* Button hover effect improves user experience */
.button:hover {
  background-color: #2980b9;
  transform: scale(1.05);
  transition: all 0.3s ease;
}

6. Accessibility Improvements

CSS can make content more accessible:

/* High contrast mode for better readability */
.high-contrast {
  background-color: black;
  color: white;
}

/* Respecting user preferences */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

7. Performance Optimization

CSS can improve perceived performance:

  • Loading styles separately allows content to be displayed faster
  • CSS sprites reduce server requests
  • Critical CSS can be inlined for above-the-fold content

History and Evolution of CSS

CSS1 (1996)

  • First official CSS specification
  • Basic styling: fonts, colors, margins, borders
  • Limited selectors and properties

CSS2 (1998)

  • Added positioning, z-index, media types
  • Introduced more selectors (child, adjacent sibling)
  • Added support for different media (print, screen)

CSS3 (1999 onwards - modular)

  • Split into modules that evolve independently
  • Introduced:
    • Selectors Level 3
    • Media Queries
    • Flexbox
    • Grid
    • Animations and Transitions
    • Gradients
    • Web Fonts
    • Rounded corners, shadows, transformations

CSS4 (ongoing)

  • Not a single version but ongoing module updates
  • New selectors (:has(), :is(), :where())
  • Container queries
  • Cascade layers
  • More advanced features

How CSS Works with HTML

Understanding the relationship between HTML and CSS is crucial:

The Rendering Process

  1. Browser loads HTML - Downloads the HTML document
  2. HTML is parsed - Browser creates the DOM (Document Object Model)
  3. CSS is loaded and parsed - Browser creates the CSSOM (CSS Object Model)
  4. Render Tree is constructed - DOM + CSSOM combined
  5. Layout - Elements are positioned on the page
  6. Paint - Elements are visually displayed

The Cascade Algorithm

The "C" in CSS stands for Cascading, which refers to how styles are applied when multiple rules conflict. The cascade follows this order:

  1. Importance - !important overrides normal declarations
  2. Specificity - More specific selectors override general ones
  3. Source Order - Later rules override earlier ones (within same specificity)
/* Example of cascade in action */
p {
  color: blue; /* Would be overridden by later rule */
}

p {
  color: red; /* This wins due to source order */
}

#special-paragraph {
  color: green; /* This wins due to higher specificity */
}

p {
  color: purple !important; /* This wins due to importance */
}

The Separation of Concerns Principle

Separation of concerns is a design principle that separates a program into distinct sections, each addressing a separate concern. In web development:

Three Layers of Web Development

  1. Content Layer (HTML)
<article>
  <h1>Article Title</h1>
  <p>Article content goes here...</p>
  <button>Read More</button>
</article>
  1. Presentation Layer (CSS)
article {
  max-width: 800px;
  margin: 0 auto;
  padding: 20px;
}

h1 {
  color: #333;
  font-size: 2em;
}

button {
  background-color: #007bff;
  color: white;
  padding: 10px 20px;
}
  1. Behavior Layer (JavaScript)
document.querySelector('button').addEventListener('click', function() {
  // Load more content
});

Benefits of Separation

Aspect Without CSS (Old Way) With CSS (Modern Way)
HTML <font color="red">Text</font> <p class="warning">Text</p>
Styling Mixed with content Separate CSS file
Updates Change every page Change one file
Readability Cluttered HTML Clean, semantic HTML

2. CSS SYNTAX

Basic Syntax Structure

CSS has a simple and straightforward syntax. Every CSS rule follows the same basic pattern:

selector {
  property: value;
  another-property: value;
}

Let's break this down:

Visual Representation

      selector
         ↓
      h1 {
    →   color: blue;    ← declaration
 prop ↑   font-size: 20px;  ↑ value
      }

Anatomy of a CSS Rule

/* This entire block is a CSS rule */
p {                           /* Opening brace */
  color: red;                 /* Declaration */
  font-size: 16px;            /* Declaration */
  line-height: 1.5;           /* Declaration */
  margin-bottom: 10px;        /* Declaration */
}                             /* Closing brace */

Understanding Selectors, Properties, and Values

1. Selectors

Selectors are patterns used to select the HTML elements you want to style.

/* Element selector */
h1 { }

/* Class selector */
.highlight { }

/* ID selector */
#header { }

/* Multiple selectors */
h1, h2, h3 { }

2. Properties

Properties are the styling aspects you want to change. They are predefined words in CSS.

/* Common properties */
color           /* Text color */
font-size       /* Size of text */
background      /* Background color or image */
margin          /* Space outside element */
padding         /* Space inside element */
border          /* Border around element */
display         /* How element is displayed */
position        /* Positioning method */

3. Values

Values specify the exact styling for a property. Different properties accept different types of values.

/* Color values */
color: red;                    /* Keyword */
color: #ff0000;                /* Hexadecimal */
color: rgb(255, 0, 0);         /* RGB */
color: rgba(255, 0, 0, 0.5);   /* RGB with opacity */
color: hsl(0, 100%, 50%);      /* HSL */

/* Length values */
font-size: 16px;               /* Pixels */
font-size: 1.2em;              /* Relative to parent */
width: 50%;                    /* Percentage */
height: 100vh;                 /* Viewport height */
margin: 10px;                  /* Pixels */

/* Keyword values */
display: block;                /* Block-level element */
display: inline;               /* Inline element */
position: relative;            /* Relative positioning */
float: left;                   /* Float left */

/* Multiple values */
margin: 10px 20px;             /* Top/bottom, left/right */
border: 1px solid black;       /* Width, style, color */
font: bold 16px Arial;         /* Weight, size, family */

CSS Rulesets

A ruleset (or rule) is the combination of a selector and its declarations.

Complete Ruleset Example

/* This entire block is a ruleset */
.card {
  /* Declaration block starts */
  width: 300px;
  padding: 20px;
  border-radius: 5px;
  background-color: white;
  box-shadow: 0 2px 5px rgba(0,0,0,0.1);
  /* Declaration block ends */
}

Multiple Rulesets

/* Multiple rulesets can be combined */

  font-family: Arial, sans-serif;
  line-height: 1.6;
  color: #333;

.container {
  max-width: 1200px;

Single-Line Comments

3. METHODS OF IMPLEMENTING CSS

There are three primary ways to apply CSS to HTML documents. Each method has its own use cases, advantages, and disadvantages.

1. Inline CSS

Inline CSS is applied directly to HTML elements using the style attribute.

Syntax

<element style="property: value; property: value;">Content</element>

Examples

<!-- Basic inline styling -->
<p style="color: blue; font-size: 16px;">This is blue text.</p>

<!-- Multiple properties -->
<div style="
  background-color: #f0f0f0;
  padding: 20px;
  border-radius: 5px;
  border: 1px solid #ccc;
">
  <h1 style="color: #333; margin-top: 0;">Inline CSS Example</h1>
  <p style="color: #666; line-height: 1.5;">
    This paragraph is styled with inline CSS.
  </p>
</div>

<!-- Dynamic styling with JavaScript -->
<button 
  onclick="this.style.backgroundColor='#4CAF50'; this.style.color='white';"
  style="padding: 10px 20px; border: none; cursor: pointer;"
>
  Click Me
</button>

Use Cases for Inline CSS

  1. Email Templates
<!-- Email clients often strip external CSS -->
<table style="width: 100%; max-width: 600px; margin: 0 auto;">
  <tr>
    <td style="padding: 20px; background-color: #f8f8f8;">
      <h1 style="color: #333;">Welcome!</h1>
    </td>
  </tr>
</table>
  1. Dynamic Styling with JavaScript
<div id="message" style="display: none;">Error message</div>
<script>
  // Show message when needed
  document.getElementById('message').style.display = 'block';
</script>
  1. Testing or Quick Prototypes
<!-- Quick style test without creating CSS files -->
<p style="color: red; font-weight: bold;">Test this style!</p>
  1. CMS Content where CSS files can't be modified
<!-- Override styles in limited environments -->
<div style="margin: 20px 0; padding: 15px; background: #ffd700;">
  Important announcement
</div>

Pros and Cons of Inline CSS

Pros Cons
Highest specificity (overrides other styles) No separation of concerns
Quick to apply Cannot reuse styles
Useful for testing Increases HTML file size
No additional HTTP requests Harder to maintain
Good for email templates No media queries
JavaScript friendly Pseudo-classes (:hover) don't work
Overrides external styles No animations or keyframes

2. Internal (Embedded) CSS

Internal CSS is placed within the <style> tag in the <head> section of an HTML document.

Syntax

<!DOCTYPE html>
<html>
<head>
  <style>
    selector {
      property: value;
      property: value;
    }
  </style>
</head>
<body>
  <!-- HTML content -->
</body>
</html>

3. External CSS

External CSS is stored in separate .css files and linked to HTML documents using the <link> tag.

Syntax

HTML file (index.html):

<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet" type="text/css" href="styles.css">
</head>
<body>
  <!-- HTML content -->
</body>
</html>

CSS file (styles.css):

/* All your CSS rules go here */
body {
  font-family: Arial, sans-serif;
  margin: 0;
  padding: 0;
}

.container {
  max-width: 1200px;
  margin: 0 auto;
}

Multiple External Stylesheets

<!-- Linking multiple CSS files -->
<head>
  <!-- Reset or normalize CSS -->
  <link rel="stylesheet" href="normalize.css">
  
  <!-- Main stylesheet -->
  <link rel="stylesheet" href="main.css">
  
  <!-- Responsive styles -->
  <link rel="stylesheet" href="responsive.css">
  
  <!-- Print styles -->
  <link rel="stylesheet" href="print.css" media="print">
  
  <!-- Conditional styles for specific pages -->
  <link rel="stylesheet" href="homepage.css">
</head>

Advanced External CSS Techniques

1. Importing CSS within CSS

/* main.css */
@import url('reset.css');
@import url('typography.css');
@import url('layout.css');

/* Your main styles */
body {
  background-color: #f4f4f4;
}

2. Media-Specific Stylesheets

<!-- Different styles for different devices -->
<link rel="stylesheet" href="desktop.css" media="screen and (min-width: 1024px)">
<link rel="stylesheet" href="tablet.css" media="screen and (max-width: 1023px)">
<link rel="stylesheet" href="mobile.css" media="screen and (max-width: 767px)">
<link rel="stylesheet" href="print.css" media="print">

3. Alternate Stylesheets

<!-- User-selectable alternate styles -->
<link rel="stylesheet" href="default.css" title="Default Style">
<link rel="alternate stylesheet" href="dark.css" title="Dark Theme">
<link rel="alternate stylesheet" href="high-contrast.css" title="High Contrast">

4. Preloading Critical CSS

<!-- Preload critical CSS for better performance -->
<link rel="preload" href="critical.css" as="style">
<link rel="stylesheet" href="critical.css">

<!-- Defer non-critical CSS -->
<link rel="preload" href="non-critical.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="non-critical.css"></noscript>

4. CSS SELECTORS - DEEP DIVE

What are Selectors?

Selectors are patterns used to select and target HTML elements for styling. They are the bridge between your HTML content and CSS styling rules.

The Role of Selectors

/* Selector targets specific elements */
selector {
  /* Styles applied to selected elements */
  property: value;
}

Think of selectors as:

  • Addresses that locate specific elements in your HTML
  • Filters that separate which elements get which styles
  • Labels that identify elements by their characteristics

How Browsers Interpret Selectors

  1. Browser reads HTML and builds DOM tree
  2. Browser reads CSS and parses selectors
  3. Browser matches selectors to DOM elements
  4. Styles are applied based on matches
HTML DOM Tree:              CSS Selectors:
html                        html { }
├── head                    head { }
├── body                    body { }
    ├── header              header { }
    ├── main                main { }
    │   ├── article         article { }
    │   └── aside           aside { }
    └── footer              footer { }

Selector Specificity

Specificity determines which CSS rule is applied when multiple rules target the same element.

Specificity Hierarchy

  1. Inline styles - Highest (1000 points)
  2. ID selectors - Very high (100 points)
  3. Class selectors - Medium (10 points)
  4. Element selectors - Low (1 point)

Specificity Calculation

/* Specificity: 0,0,1 (1 point) */
p { color: blue; }

/* Specificity: 0,1,0 (10 points) */
.text { color: green; }

/* Specificity: 1,0,0 (100 points) */
#main { color: red; }

/* Specificity: 1,1,1 (111 points) */
#main p.text { color: purple; }

Specificity Examples

<p id="special" class="highlight">What color will this be?</p>

/* Rule 1: Specificity 0,1,0 (10) */
.highlight { color: blue; }

/* Rule 2: Specificity 1,0,0 (100) */
#special { color: red; }

/* Rule 3: Specificity 0,0,1 (1) */
p { color: green; }

/* Result: Red wins due to highest specificity */

Specificity Chart

Selector Pattern Example Specificity
Inline style style="color: red;" 1000
ID selector #header 100
Class selector .menu 10
Attribute selector [type="text"] 10
Pseudo-class :hover 10
Element selector div 1
Pseudo-element ::before 1
Universal * 0

Important Notes on Specificity

  1. !important overrides everything

    p {
  color: red !important; /* Overrides all */
}

  2. Equal specificity: last one wins

    .text { color: blue; }
.text { color: green; } /* This wins */

  3. Context doesn't affect specificity

    /* Both have same specificity (0,0,1) */
p { color: blue; }
body p { color: red; } /* Still 0,0,1? No! */

    Actually, body p has specificity 0,0,2 (2 element selectors)

Selector Performance

Different selectors have different performance characteristics.

Performance Hierarchy (Fastest to Slowest)

  1. ID selectors - #header
  2. Element selectors - div
  3. Class selectors - .menu
  4. Adjacent selectors - div + p
  5. Child selectors - ul > li
  6. Descendant selectors - div span
  7. Universal selectors - *
  8. Attribute selectors - [type="text"]
  9. Pseudo-classes - :nth-child()

Performance Best Practices

/* GOOD - Simple and fast */
#sidebar { }
.nav-item { }
h2 { }

/* BAD - Overly complex and slow */
* div > ul li:first-child + .item { }

/* ACCEPTABLE - Complex but necessary */
.sidebar .nav-menu li.active > a { }

Modern Browsers and Performance

Modern browsers are extremely fast at matching selectors. Focus on:

  • Maintainability over micro-optimizations
  • Specificity management
  • Readability for team collaboration
  • Avoiding overly complex selectors

Benefits of Grouping Selectors

  1. Reduces Code Duplication

    /* Without grouping - repetitive */
h1 { font-family: Arial; }
h2 { font-family: Arial; }
h3 { font-family: Arial; }

/* With grouping - DRY */
h1, h2, h3 { font-family: Arial; }

  2. Easier Maintenance

    /* Change once, affects all */
h1, h2, h3, .title {
  color: #333;
}

  3. Cleaner Code

    /* All form elements share base styles */
input, textarea, select {
  border: 1px solid #ddd;
  padding: 8px;
  border-radius: 4px;
}

7. COMMON MISTAKES AND HOW TO AVOID THEM

Selector Errors

Mistake 1: Overly Specific Selectors

BAD:

body div.container #main-content article p.intro {
  color: blue;
}

GOOD:

.intro {
  color: blue;
}
/* OR */
.container .intro {
  color: blue;
}

Mistake 2: Using IDs for Reusable Styles

BAD:

#button {
  padding: 10px 20px;
  background-color: blue;
}
/* Can't reuse this style */

GOOD:

.btn {
  padding: 10px 20px;
  background-color: blue;
}
/* Reusable across multiple elements */

Mistake 3: Not Using Classes When Needed

BAD:

header div ul li a {
  color: blue;
}
/* Fragile - breaks if HTML structure changes */

GOOD:

.nav-link {
  color: blue;
}
/* Robust - independent of HTML structure */

Implementation Errors

Mistake 4: Inline CSS Overuse

BAD:

<p style="color: red; font-size: 16px; margin: 10px;">Text</p>
<p style="color: red; font-size: 16px; margin: 10px;">Another text</p>

GOOD:

<style>
  .highlighted-text {
    color: red;
    font-size: 16px;
    margin: 10px;
  }
</style>
<p class="highlighted-text">Text</p>
<p class="highlighted-text">Another text</p>

Mistake 5: Mixing Implementation Methods

BAD:

<head>
  <style>
    p { color: red; }
  </style>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p style="color: blue;">Text</p>
</body>
<!-- Confusing - multiple sources for same element -->

GOOD:

<head>
  <!-- Stick to one primary method -->
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p>Text</p>
</body>

Mistake 6: Forgetting the Cascade

BAD:

/* styles.css */
p {
  color: red;
}

/* later in same file or another stylesheet */
p {
  color: blue;
}
/* Which one wins? (blue - later in source order) */

GOOD:

/* Be intentional about order */
/* Base styles first */
p {
  color: red;
}

/* Overrides later */
.highlight p {
  color: blue;
}

Specificity Issues

Mistake 7: Overusing !important

BAD:

.text {
  color: blue !important;
}

.special-text {
  color: red; /* Won't work because of !important */
}

GOOD:

/* Use specificity instead of !important */
.text {
  color: blue;
}

.special-text {
  color: red;
}

/* Or if necessary, use more specific selector */
.container .text {
  color: blue;
}

Mistake 8: Not Understanding Specificity

BAD:

/* Expecting this to work */
.nav-item {
  color: blue;
}

#main-nav .nav-item {
  color: red; /* This will override because ID is more specific */
}

GOOD:

/* Keep specificity consistent */
.nav-item {
  color: blue;
}

.main-nav .nav-item {
  color: red; /* Use class instead of ID */
}

Performance Issues

Mistake 9: Universal Selector in Key Places

BAD:

/* Applied to EVERY element on every interaction */
* {
  transition: all 0.3s ease;
}

GOOD:

/* Target only elements that need transitions */
.btn,
.nav-link,
.card {
  transition: all 0.3s ease;
}

Mistake 10: Deep Nesting

BAD:

.main-content .sidebar .widget .list li a span {
  color: red;
}
/* Hard to read, slow to match */

GOOD:

.widget-link span {
  color: red;
}
/* Simple and fast */

Syntax Errors

Mistake 11: Missing Semicolons

BAD:

p {
  color: red
  font-size: 16px
}
/* Second property won't work */

GOOD:

p {
  color: red;
  font-size: 16px;
}

9. REVIEW QUESTIONS AND ANSWERS

Section 1: CSS Basics

Q1: What does CSS stand for and what is its purpose?

A: CSS stands for Cascading Style Sheets. Its purpose is to control the presentation and layout of HTML documents. It separates content (HTML) from presentation, allowing for:

  • Consistent styling across multiple pages
  • Easier maintenance
  • Better accessibility
  • Responsive design
  • Improved user experience

Q2: What are the three ways to implement CSS? Explain each briefly.

A: The three ways to implement CSS are:

  1. Inline CSS: Applied directly to HTML elements using the style attribute 

    <p style="color: red;">Text</p>

  2. Internal CSS: Placed within <style> tags in the HTML <head>

    <style>
  p { color: red; }
</style>

  3. External CSS: Stored in separate .css files linked to HTML

    <link rel="stylesheet" href="styles.css">

Q3: What are the advantages and disadvantages of external CSS?

A:

Advantages:

  • Reusable across multiple pages
  • Cachable by browsers (faster loading)
  • Easier maintenance
  • Separates content from presentation
  • Better organization

Disadvantages:

  • Additional HTTP requests
  • Pages may render unstyled initially (FOUC)
  • Requires proper file management
  • Loading order matters

Section 2: CSS Syntax

Q4: What is the basic syntax of a CSS rule?

A: The basic syntax is:

selector {
  property: value;
  another-property: value;
}

For example:

h1 {
  color: blue;
  font-size: 20px;
}

Q5: How do you write comments in CSS?

A: CSS comments are written using /* */:

/* This is a single-line comment */

/*
  This is a

Q7: What is the universal selector and when would you use it?

A: The universal selector (*) selects all elements on the page. Common uses include:

  • CSS resets
  • Setting box-sizing: border-box on all elements
  • Debugging (temporarily outlining all elements)
  • Applying base styles globally

Example:

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

Q8: How do you group multiple selectors that share the same styles?

A: Use commas to separate selectors:

h1, h2, h3 {
  font-family: Arial, sans-serif;
  color: #333;
}

Q9: What is selector specificity and how is it calculated?

A: Specificity determines which CSS rule applies when multiple rules target the same element. It's calculated as:

  • Inline styles: 1000 points
  • ID selectors: 100 points
  • Class selectors: 10 points
  • Element selectors: 1 point

Example:

p { color: blue; }           /* Specificity: 1 */
.text { color: green; }      /* Specificity: 10 */
#main { color: red; }        /* Specificity: 100 */

Section 4: Practical Application

Q10: When would you use inline CSS?

A: Inline CSS is appropriate for:

  • HTML email templates
  • Quick testing or prototyping
  • Dynamic styling with JavaScript
  • CMS environments with limited CSS access
  • Overriding other styles when necessary

Q11: What's the best practice for organizing CSS in a large project?

A: Best practices include:

  • Use external CSS files
  • Organize by component/module
  • Follow a naming convention (BEM, etc.)
  • Use comments to document sections
  • Keep specificity low
  • Avoid !important
  • Group related properties
  • Use a preprocessor (Sass/Less) for large projects

Q12: What happens if two rules have the same specificity?

A: When two rules have the same specificity, the one that appears later in the source code wins (cascade order).

p { color: blue; }
p { color: red; } /* This wins */

Section 5: Common Mistakes

Q13: Why should you avoid overly specific selectors?

A: Overly specific selectors:

  • Are hard to maintain
  • Make overriding styles difficult
  • Decrease performance
  • Create tight coupling with HTML structure
  • Reduce reusability

BAD: body div.container #main-content article p.intro
GOOD: .intro

Q14: What's wrong with using !important frequently?

A: Overusing !important:

  • Breaks the natural cascade
  • Makes debugging difficult
  • Creates specificity wars
  • Reduces maintainability
  • Makes overrides nearly impossible

Instead, use proper specificity and source order.

Q15: How can you avoid CSS conflicts in large projects?

A: To avoid conflicts:

  • Use a naming convention (BEM)
  • Keep selectors simple
  • Avoid IDs for styling
  • Use specific parent containers
  • Organize CSS by component
  • Use CSS methodologies (OOCSS, SMACSS)
  • Consider CSS-in-JS for component-based apps

Section 6: Advanced Concepts

Q16: Can you combine multiple class selectors?

A: Yes, elements can have multiple classes:

<p class="highlight warning important">Text</p>

Styles from all classes are combined. If properties conflict, later classes in the CSS override earlier ones.

Q17: How do pseudo-classes differ from regular classes?

A: Pseudo-classes target elements based on their state or position, not their class attribute:

/* Regular class */
.highlight { }

/* Pseudo-class */
a:hover { }           /* Hover state */
li:first-child { }    /* Position */
input:focus { }       /* Focus state */

Q18: What is the cascade in CSS?

A: The cascade is the algorithm that determines which styles apply when multiple rules conflict. It considers:

  1. Importance (!important)
  2. Specificity
  3. Source order
  4. Inheritance
