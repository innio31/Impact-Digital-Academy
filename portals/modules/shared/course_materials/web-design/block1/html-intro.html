1. INTRODUCTION TO HTML

1.1 What is HTML?

HTML (HyperText Markup Language) is the standard markup language used to create web pages. It forms the structural foundation of every website on the internet. Think of HTML as the skeleton or framework of a building—it provides the structure upon which everything else is built.

1.2 A Brief History of HTML

Version Year Key Features
HTML 1.0 1991 Basic tags, no tables or forms
HTML 2.0 1995 Standardized core features
HTML 3.2 1997 Tables, applets, text flow
HTML 4.01 1999 Cascading Style Sheets (CSS) support
XHTML 1.0 2000 Stricter, XML-based syntax
HTML5 2014 Semantic elements, multimedia, APIs

HTML5 is the current standard and brings:

  • Native audio and video support
  • Canvas for drawing graphics
  • Semantic elements (article, section, nav)
  • Better form controls
  • Improved accessibility features

1.3 How HTML Works with Other Technologies

┌─────────────────────────────────────┐
│         CONTENT (HTML)              │
│  "This is a heading. This is text." │
├─────────────────────────────────────┤
│         PRESENTATION (CSS)           │
│  "Make the heading blue and big.     │

├─────────────────────────────────────┤
│       INTERACTIVITY (JavaScript)     │

└─────────────────────────────────────┘
  • HTML: Structure and content (nouns)
  • CSS: Styling and layout (adjectives)
  • JavaScript: Behavior and interactivity (verbs)

2. THE ROLE OF HTML IN WEB DEVELOPMENT

2.1 Primary Functions of HTML

2.1.1 Content Structuring

HTML organizes content into a logical hierarchy, making it understandable for both humans and machines.

Example:

<article>
    <h1>Breaking News: Major Discovery</h1>
    <p>Scientists have discovered a new species in the Amazon rainforest.</p>
    <p>The discovery was made during a three-year expedition.</p>
</article>

2.1.2 Semantic Meaning

HTML gives meaning to content, which helps:

  • Search Engines: Understand what your content is about (SEO)
  • Screen Readers: Help visually impaired users navigate
  • Browsers: Render content appropriately
  • Developers: Maintain and understand code

2.1.3 Accessibility Foundation

Proper HTML ensures web content is accessible to everyone, including people with disabilities.

Accessible vs. Inaccessible Example:

<!-- Inaccessible - no meaning -->
<div class="big-text">Welcome</div>
<div class="small-text">Click here</div>

<!-- Accessible - semantic meaning -->
<h1>Welcome</h1>
<button>Click here</button>

2.2 HTML in the Web Ecosystem

User Types URL → Browser Requests Page → Server Sends HTML
         ↓
Browser Receives HTML
         ↓
Browser Parses HTML (creates DOM)
         ↓
Browser Requests CSS, JavaScript, Images
         ↓
Browser Renders Complete Page

3. BASIC HTML DOCUMENT STRUCTURE

3.1 The Minimal HTML5 Document

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first paragraph.</p>
</body>
</html>

3.2 Detailed Breakdown of Each Component

3.2.1 The DOCTYPE Declaration

<!DOCTYPE html>

Purpose: Tells the browser which version of HTML the page is written in.

Key Points:

  • Must be the very first line of every HTML document
  • Not case-sensitive, but typically written in uppercase
  • For HTML5, it's simple and easy to remember
  • Without it, browsers may render pages in "quirks mode" (inconsistent rendering)

Historical Context:

<!-- HTML 4.01 Strict -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

<!-- XHTML 1.0 Strict -->
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

<!-- HTML5 (current standard) -->
<!DOCTYPE html>

3.2.2 The HTML Element

<html lang="en">
    <!-- All content goes here -->
</html>

Purpose: The root element that wraps all content on the entire page.

Important Attributes:

  • lang: Specifies the language of the document (important for accessibility and search engines)
    • en = English
    • es = Spanish
    • fr = French
    • ar = Arabic

Best Practice: Always include the lang attribute for accessibility.

3.2.3 The HEAD Element

<head>
    <meta charset="UTF-8">
    <meta name="description" content="Free web tutorials">
    <meta name="keywords" content="HTML, CSS, JavaScript">
    <meta name="author" content="John Doe">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Website - Home Page</title>
    <link rel="stylesheet" href="styles.css">
    <script src="script.js" defer></script>
</head>

Purpose: Contains meta-information about the document—data that describes the data.

What Goes in the Head:

Element Purpose Example
<title> Page title (appears in browser tab) <title>Home | My Site</title>
<meta> Metadata (character set, description, etc.) <meta charset="UTF-8">
<link> Links to external resources (CSS, icons) <link rel="stylesheet" href="style.css">
<style> Internal CSS styles <style>body {color: blue;}</style>
<script> JavaScript code or links <script src="app.js"></script>
<base> Base URL for relative links <base href="https://example.com/">

Critical Meta Tags Explained:

<!-- Character encoding - should be UTF-8 for universal character support -->
<meta charset="UTF-8">

<!-- Viewport settings for responsive design on mobile devices -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- SEO: Description shown in search results (150-160 characters) -->
<meta name="description" content="Learn HTML basics with our comprehensive guide. Perfect for beginners starting their web development journey.">

<!-- SEO: Keywords (less important now, but still used) -->
<meta name="keywords" content="HTML, tutorial, beginners, web development">

<!-- Author information -->
<meta name="author" content="Your Name">

<!-- Refresh/redirect (use with caution!) -->
<meta http-equiv="refresh" content="30"> <!-- Refresh every 30 seconds -->

3.2.4 The BODY Element

<body>
    <!-- All visible content goes here -->
    <header>
        <h1>Welcome to My Website</h1>
        <nav>
            <a href="/">Home</a>
            <a href="/about">About</a>
        </nav>
    </header>
    
    <main>
        <article>
            <h2>Blog Post Title</h2>
            <p>Blog post content...</p>
        </article>
    </main>
    
    <footer>
        <p>&copy; 2024 My Website</p>
    </footer>
</body>

Purpose: Contains all the visible content of a web page—everything that users see when they visit your site.

What Can Go in the Body:

  • Text content (headings, paragraphs, lists)
  • Multimedia (images, videos, audio)
  • Forms for user input
  • Interactive elements (buttons, links)
  • Structural elements (headers, footers, sections)

4. UNDERSTANDING ELEMENTS AND TAGS

4.1 What Are HTML Tags?

Tags are the hidden keywords within a web page that define how the browser must format and display the content.

Tag Syntax:

<opening-tag attribute="value">Content</closing-tag>

Types of Tags:

Type Format Example
Paired Tags <tag>content</tag> <p>This is a paragraph.</p>
Self-closing Tags <tag> or <tag /> <img src="image.jpg" alt="description">
Void Elements Elements with no content <br>, <hr>, <input>

4.2 Anatomy of an HTML Element

┌─────────────┐
│   <p class="intro">Hello, world!</p>   │
└─────────────┘
  ↑            ↑                   ↑
  │            │                   │
Opening      Attribute           Closing
Tag          with Value            Tag
              │
           Content

4.3 Element Categories

4.3.1 Block-level Elements

  • Create "blocks" of content
  • Start on a new line
  • Take up the full width available
<div>
    <h1>This is a block-level heading</h1>
    <p>This paragraph is also block-level and starts on a new line.</p>
    <p>Another paragraph, notice how each paragraph is on its own line.</p>
</div>

Common Block Elements:

  • <div> - Generic container
  • <h1> to <h6> - Headings
  • <p> - Paragraphs
  • <ul>, <ol>, <li> - Lists
  • <header>, <footer>, <section> - Semantic containers

4.3.2 Inline Elements

  • Do not start on a new line
  • Only take up as much width as necessary
  • Flow within block elements
<p>This is a paragraph with <strong>inline</strong> elements like 
   <a href="#">links</a>, <em>emphasized text</em>, and 
   <span class="highlight">spans</span>.</p>

Common Inline Elements:

  • <span> - Generic inline container
  • <a> - Links
  • <strong>, <em> - Text emphasis
  • <img> - Images
  • <button> - Buttons

4.3.3 Empty (Void) Elements

  • Have no content
  • Cannot have closing tags
  • Can have attributes
<!-- Line break -->
<br>

<!-- Horizontal rule -->
<hr>

<!-- Image (self-closing syntax) -->
<img src="photo.jpg" alt="Description">

<!-- Input field -->
<input type="text" name="username">

4.4 Element Relationships

Understanding how elements relate to each other is crucial for CSS and JavaScript.

<html>
    <head>           <!-- Child of html -->
        <title>      <!-- Child of head, descendant of html -->
            Page Title  <!-- Text node, descendant of title -->
        </title>
    </head>
    <body>           <!-- Child of html -->
        <h1>         <!-- Child of body, descendant of html -->
            Heading  <!-- Text node, descendant of h1 -->
        </h1>
        <p>          <!-- Sibling of h1, child of body -->
            Text     <!-- Text node, descendant of p -->
        </p>
    </body>
</html>

Relationship Terminology:

  • Parent: The direct container of an element (e.g., body is parent of h1)
  • Child: An element directly inside another (e.g., h1 is child of body)
  • Sibling: Elements sharing the same parent (e.g., h1 and p are siblings)
  • Ancestor: Any element that contains another (e.g., html is ancestor of p)
  • Descendant: Any element inside another (e.g., p is descendant of html)

4.5 HTML Attributes

Attributes provide additional information about HTML elements.

Basic Syntax:

<element attribute="value">content</element>

4.5.1 Global Attributes (Work with any element)

Attribute Purpose Example
class For CSS styling and JavaScript selection <p class="highlight error">
id Unique identifier for an element <div id="main-content">
style Inline CSS styling <p style="color: blue;">
title Tooltip text on hover <abbr title="HyperText Markup Language">HTML</abbr>
lang Language of element content <p lang="es">Hola</p>
dir Text direction <p dir="rtl">نص عربي</p>
data-* Custom data attributes <div data-user-id="12345">
tabindex Keyboard navigation order <button tabindex="1">
hidden Hide an element <div hidden>Secret message</div>

4.5.2 Event Handler Attributes (for JavaScript)

<button onclick="alert('Clicked!')">Click Me</button>
<input onchange="updateValue(this.value)">
<form onsubmit="return validateForm()">
<div onmouseover="highlight(this)">

Common event attributes:

  • onclick - When element is clicked
  • onload - When page/element finishes loading
  • onmouseover - When mouse moves over element
  • onchange - When form input changes
  • onsubmit - When form is submitted

5. HEADINGS IN HTML (H1 TO H6)

5.1 The Heading Hierarchy

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important). 

<h1>Main Page Title (Most Important)</h1>
<h2>Section Heading</h2>
<h3>Sub-section Heading</h3>
<h4>Sub-sub-section Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading (Least Important)</h6>

5.2 Visual Representation of Heading Sizes

┌────────────────────────────────────┐
│ h1: Main Title (largest)           │ 32px (default)
├────────────────────────────────────┤
│   h2: Section Title                │ 24px
├────────────────────────────────────┤
│       h3: Sub-section               │ 20px
├────────────────────────────────────┤
│           h4: Minor heading         │ 18px
├────────────────────────────────────┤
│               h5: Small heading     │ 16px
├────────────────────────────────────┤
│                   h6: Tiny heading  │ 14px (smallest)
└────────────────────────────────────┘

Note: These sizes are browser defaults and can be changed with CSS.

5.3 Proper Heading Structure

5.3.1 Correct Hierarchy (Good)

<article>
    <h1>The Complete Guide to Coffee Brewing</h1>
    
    <h2>Introduction to Coffee</h2>
    <p>Coffee is a brewed drink prepared from roasted coffee beans...</p>
    
    <h2>Brewing Methods</h2>
    
    <h3>Pour-Over Method</h3>
    <p>The pour-over method involves...</p>
    
    <h3>French Press</h3>
    <p>The French press method...</p>
    
    <h2>Choosing Coffee Beans</h2>
    
    <h3>Arabica vs Robusta</h3>
    <p>Arabica beans are known for...</p>
    
    <h4>Arabica Varieties</h4>
    <p>Common Arabica varieties include...</p>
</article>

5.3.2 Incorrect Hierarchy (Bad)

<article>
    <h3>Wrong - Starting with h3 skips levels</h3>
    
    <h5>Even worse - inconsistent hierarchy</h5>
    
    <h2>This creates confusion about importance</h2>
    
    <h1>Multiple h1 elements are also problematic</h1>
</article>

5.4 SEO and Accessibility Importance

5.4.1 For Search Engines

Search engines use headings to understand your content structure:

Google's Perspective:
┌─────────────────────────────────┐
│ <h1>Best Coffee Shops in NYC</h1>│ ← Main topic
├─────────────────────────────────┤
│   <h2>Manhattan</h2>            │ ← Subtopic
│   <h2>Brooklyn</h2>             │ ← Subtopic
│       <h3>Williamsburg</h3>     │ ← Sub-subtopic
│       <h3>DUMBO</h3>            │ ← Sub-subtopic
└─────────────────────────────────┘

SEO Best Practices:

  • Only one <h1> per page
  • Don't skip heading levels (h1 → h3 without h2 is bad)
  • Include relevant keywords in headings
  • Keep headings descriptive and concise

5.4.2 For Accessibility (Screen Readers)

Screen reader users often navigate by headings:

Screen Reader Announcement:
"Heading level 1: The Complete Guide to Coffee Brewing"
"Heading level 2: Introduction to Coffee"
"Heading level 2: Brewing Methods"
"Heading level 3: Pour-Over Method"
"Heading level 3: French Press"
"Heading level 2: Choosing Coffee Beans"

Keyboard Shortcut for Screen Reader Users:

  • Press 'H' to jump between headings
  • Press '1' to jump to next h1
  • Press '2' to jump to next h2, etc.

5.5 Styling Headings with CSS

While styling is CSS territory, it's important to understand the relationship:

<!DOCTYPE html>
<html>

</body>
</html>

6. PARAGRAPHS AND TEXT CONTENT

6.1 The Paragraph Element (<p>)

The <p> element represents a paragraph of text. It's one of the most commonly used HTML elements.

Basic Syntax:

<p>This is a paragraph of text. It can contain multiple sentences and can span multiple lines in the code, but the browser will render it as a single block of text.</p>

6.2 Paragraph Characteristics

6.2.1 Default Styling

<!DOCTYPE html>
<html>
<head>
    <style>
        /* Default browser styles for paragraphs (simplified) */
        p {
            display: block;        /* Takes full width */
            margin-top: 1em;       /* Space above */
            margin-bottom: 1em;     /* Space below */
            margin-left: 0;
            margin-right: 0;
            line-height: 1.5;       /* Space between lines */
        }
    </style>
</head>
<body>
    <p>First paragraph - notice the space above and below.</p>
    <p>Second paragraph - there's automatic spacing between paragraphs.</p>
    <p>Third paragraph - this spacing improves readability.</p>
</body>
</html>

6.2.2 Visual Output:

First paragraph - notice the space above and below. (margin)

Second paragraph - there's automatic spacing between paragraphs. (margin)

Third paragraph - this spacing improves readability. (margin)

6.3 Paragraph Best Practices

6.3.1 Keep Paragraphs Focused

Good:

<p>Coffee is a brewed drink prepared from roasted coffee beans, the seeds of berries from certain Coffea species. The genus Coffea is native to tropical Africa and Madagascar, the Comoros, Mauritius, and Réunion in the Indian Ocean.</p>

<p>Coffee plants are now cultivated in over 70 countries, primarily in the equatorial regions of the Americas, Southeast Asia, India, and Africa. The two most commonly grown are the highly regarded arabica, and the less sophisticated but stronger and more hardy robusta.</p>

Bad (Too Long):

<p>Coffee is a brewed drink prepared from roasted coffee beans, the seeds of berries from certain Coffea species. The genus Coffea is native to tropical Africa and Madagascar, the Comoros, Mauritius, and Réunion in the Indian Ocean. Coffee plants are now cultivated in over 70 countries, primarily in the equatorial regions of the Americas, Southeast Asia, India, and Africa. The two most commonly grown are the highly regarded arabica, and the less sophisticated but stronger and more hardy robusta. Coffee cultivation first took place in southern Arabia; the earliest credible evidence of coffee-drinking appears in the middle of the 15th century in the Sufi shrines of Yemen. In the Horn of Africa and Yemen, coffee was used in local religious ceremonies. As these ceremonies conflicted with the beliefs of the Christian church, the Ethiopian Church banned coffee until the reign of Emperor Menelik II. The beverage was also banned in Ottoman Turkey during the 17th century for political reasons and was associated with rebellious political activities in Europe.</p>

6.4 Line Breaks vs. Paragraphs

6.4.1 Using <br> for Line Breaks

<!-- Address example - correct use of line breaks -->
<p>
    John Doe<br>
    123 Main Street<br>
    New York, NY 10001<br>
    United States
</p>

<!-- Poetry - correct use of line breaks -->
<p>
    Roses are red,<br>
    Violets are blue,<br>
    HTML is fun,<br>
    And so are you!
</p>

6.4.2 DON'T Use <br> for Spacing

Wrong:

<p>This is the first paragraph.</p>
<br>
<br>
<br>
<p>This is way too far from the first paragraph.</p>

Correct (use CSS margin):

<p style="margin-bottom: 3em;">This is the first paragraph.</p>
<p>This paragraph has proper spacing via CSS.</p>

6.8 Accessibility Considerations for Text

6.8.1 Readable Line Length

<!-- Good - readable line length (45-75 characters) -->
<p style="max-width: 65ch;">
    Studies show that optimal line length for readability is between 
    45 and 75 characters per line. This makes text easier to read 
    and reduces eye strain. Using the 'ch' unit in CSS helps 
    maintain this ideal length.
</p>

<!-- Bad - too wide, hard to read -->
<p style="width: 100%;">
    This paragraph stretches across the entire screen, making it very difficult to read because your eyes have to travel too far from the end of one line to the beginning of the next. This causes reader fatigue and reduces comprehension.
</p>

6.8.2 Sufficient Contrast

<!-- Good contrast -->
<p style="color: #333; background: #fff;">
    This text has good contrast with its background.
</p>

<!-- Bad contrast - hard to read -->
<p style="color: #ccc; background: #fff;">
    This light gray text on white is difficult to read.
</p>

<!-- Even worse - accessibility disaster -->
<p style="color: #ff0000; background: #00ff00;">
    Red on green is nearly impossible for colorblind users to read.
</p>

8. COMMON MISTAKES AND HOW TO AVOID THEM

8.1 Document Structure Mistakes

Mistake 1: Missing DOCTYPE

<!-- BAD - No DOCTYPE -->
<html>
<head>
    <title>My Page</title>
</head>
<body>
    <p>Content</p>
</body>
</html>

<!-- GOOD - With DOCTYPE -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Page</title>
</head>
<body>
    <p>Content</p>
</body>
</html>

Why it matters: Without DOCTYPE, browsers enter "quirks mode" and render pages inconsistently.

Mistake 2: Missing Character Encoding

<!-- BAD - Special characters may not display correctly -->
<head>
    <title>My Page</title>
</head>

<!-- GOOD - UTF-8 ensures all characters display properly -->
<head>
    <meta charset="UTF-8">
    <title>My Page</title>
</head>

Mistake 3: Missing Language Attribute

<!-- BAD - Screen readers don't know which language to use -->
<html>

<!-- GOOD - Screen readers use correct pronunciation -->
<html lang="en">

8.2 Heading Mistakes

Mistake 4: Skipping Heading Levels

<!-- BAD - Jumping from h1 to h3 -->
<h1>Main Title</h1>
<h3>Sub-section</h3>  <!-- h2 is missing! -->

<!-- GOOD - Proper hierarchy -->
<h1>Main Title</h1>
<h2>Section</h2>
<h3>Sub-section</h3>

Mistake 5: Multiple H1 Elements

<!-- BAD - Multiple main headings -->
<body>
    <h1>Site Title</h1>
    <h1>Another Important Heading</h1>  <!-- Second h1 -->
</body>

<!-- GOOD - Only one h1 -->
<body>
    <h1>Site Title</h1>
    <h2>Section Heading</h2>
    <h2>Another Section</h2>
</body>

Mistake 6: Using Headings for Styling Only

<!-- BAD - Using h1 just because it's big -->
<p>Welcome to our site!</p>
<h1>Contact Information</h1>  <!-- This should be an h2 if it's a section -->
<p>Email: info@example.com</p>

<!-- GOOD - Using appropriate heading levels -->
<h1>Welcome to Our Site</h1>
<h2>Contact Information</h2>
<p>Email: info@example.com</p>

8.3 Paragraph Mistakes

Mistake 7: Using <br> for Paragraph Spacing

<!-- BAD - Using line breaks for spacing -->
<br>
<br>
<p>This is a paragraph.</p>
<br>
<br>
<p>This is another paragraph.</p>

<!-- GOOD - Let paragraphs do their job -->
<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

<!-- Or use CSS for custom spacing -->
<p style="margin-bottom: 2em;">First paragraph</p>
<p>Second paragraph with spacing above via CSS.</p>

Mistake 8: Nesting Block Elements in Paragraphs

<!-- BAD - Can't put block elements inside p -->
<p>
    Some text here.
    <h2>This is invalid</h2>  <!-- h2 cannot be inside p -->
    More text.
</p>

<!-- GOOD - Close p before starting new block -->
<p>Some text here.</p>
<h2>This is correct</h2>
<p>More text.</p>

Mistake 9: Empty Paragraphs for Spacing

<!-- BAD - Empty paragraphs -->
<p></p>
<p></p>
<p></p>
<p>Real content here</p>

<!-- GOOD - Use CSS margin instead -->
<p style="margin-top: 3em;">Real content here</p>

8.4 Accessibility Mistakes

Mistake 10: Missing Alt Text on Images

<!-- BAD - No alt text -->
<img src="logo.png">

<!-- GOOD - Descriptive alt text -->
<img src="logo.png" alt="Company Name Logo">

<!-- For decorative images, empty alt is acceptable -->
<img src="decorative-line.png" alt="" role="presentation">

Mistake 11: Using Improper Semantic Elements

<!-- BAD - All divs, no structure -->
<div class="header">
    <div class="title">Site Title</div>
</div>
<div class="main">
    <div class="article">Content</div>
</div>

<!-- GOOD - Semantic HTML5 elements -->
<header>
    <h1>Site Title</h1>
</header>
<main>
    <article>
        <h2>Article Title</h2>
        <p>Content</p>
    </article>
</main>

Mistake 12: Poor Link Text

<!-- BAD - Uninformative link text -->
<p>For more information, <a href="/click-here">click here</a>.</p>

<!-- GOOD - Descriptive link text -->
<p>Read our <a href="/guide">complete guide to hiking</a> for more information.</p>

<!-- Or better yet -->
<p><a href="/guide">Read our complete guide to hiking</a> for tips and advice.</p>

8.5 Validation and Best Practices Checklist

Use this checklist to ensure your HTML is correct:

□ DOCTYPE declaration present at the top
□ html tag has lang attribute
□ meta charset="UTF-8" in head
□ title element is present and descriptive
□ Only one h1 per page
□ Heading hierarchy is logical (no skipped levels)
□ All images have alt attributes
□ All form inputs have associated labels
□ Links have descriptive text (not "click here")
□ No block elements inside inline elements (like p)
□ All tags are properly closed/nested
□ Use semantic elements where appropriate
□ Indentation is consistent for readability
□ No inline styles (use CSS classes instead)
□ All special characters are escaped (&amp; for &)

9.4 Self-Assessment Questions

Question 1: What is the purpose of the <!DOCTYPE html> declaration?

Answer: It tells the browser which version of HTML the page is using. For HTML5, it ensures the browser renders the page in standards mode, not quirks mode.

Question 2: Why should there only be one <h1> per page?

Answer: The h1 represents the main heading/topic of the entire page. Having multiple h1 elements confuses search engines and screen readers about the primary content of the page.

Question 3: What's the difference between a block-level element and an inline element?

Answer: Block-level elements (like p and h1) start on a new line and take up the full width available. Inline elements (like strong and a) flow within the text and only take up as much width as necessary.

Question 4: Why is it important to use semantic HTML elements (like article, section, nav) instead of just divs?

Answer: Semantic elements provide meaning to the content structure. They help:

  • Search engines understand the page better
  • Screen readers navigate the content
  • Other developers understand your code
  • Browsers provide default styling where appropriate

Question 5: What information should go in the <head> section versus the <body>?

Answer: The head contains meta-information about the document (title, character set, CSS links, etc.). The body contains all visible content that users see on the page.

CONCLUSION

HTML is the foundation of all web development. Mastering these basics—document structure, elements, tags, headings, and paragraphs—gives you the power to create well-structured, accessible, and semantic web pages. Remember:

  • Always include the DOCTYPE and character encoding
  • Use semantic HTML elements appropriately
  • Structure your headings logically (h1 → h6)
  • Keep paragraphs focused on single ideas
  • Test your code with validators and accessibility tools
  • Practice regularly to reinforce your learning

The web is built on HTML. By understanding these fundamentals, you're taking the first step toward becoming a proficient web developer. Keep coding, keep learning, and don't hesitate to refer to the resources provided when you need help.
