Introduction

Deploying a React application involves more than just uploading files to a server. It requires careful preparation to ensure your application is secure, performant, and properly configured for the production environment. This guide covers three critical aspects of deployment preparation: managing environment variables, optimizing your React app for production, and choosing the right hosting platform.

Part 1: Environment Variables for Hiding API Keys

What Are Environment Variables?

Environment variables are dynamic named values that can affect the way running processes behave on a computer. In web development, they are used to store configuration settings that change between environments (development, staging, production) without modifying the code.

Why Are They Important?

  1. Security: Keep sensitive information like API keys, database URLs, and authentication secrets out of your codebase
  2. Flexibility: Different environments can use different configurations without code changes
  3. Compliance: Prevents accidental exposure of credentials in version control systems

Common Sensitive Data to Protect

  • API Keys (Google Maps, Stripe, Firebase, etc.)
  • Database connection strings
  • Authentication secrets
  • Third-party service credentials
  • Environment-specific URLs

Implementing Environment Variables in React

Step 1: Create Environment Files

React (created with Vite or Create React App) supports environment variables through special files:

Step 2: Define Your Variables

In .env.development:

VITE_API_URL=http://localhost:3000/api
VITE_GOOGLE_MAPS_KEY=dev_abc123xyz

In .env.production:

VITE_API_URL=https://api.myapp.com
VITE_GOOGLE_MAPS_KEY=prod_def456uvw

Note for Vite: Variables must be prefixed with VITE_ to be exposed to the client-side code.

Note for Create React App: Variables must be prefixed with REACT_APP_.

Step 3: Access Variables in Your Code

// Using environment variables in React components
const API_URL = import.meta.env.VITE_API_URL; // Vite syntax
// const API_URL = process.env.REACT_APP_API_URL; // CRA syntax

function App() {
  const [data, setData] = useState(null);
  
  useEffect(() => {
    fetch(`${API_URL}/users`)
      .then(response => response.json())
      .then(data => setData(data));
  }, []);
  
  return (
    <div>
      <h1>My App</h1>
      {/* Component content */}
    </div>
  );
}

Best Practices for Environment Variables

  1. Never commit real secrets to version control
  2. Use different keys for development and production
  3. Validate variables at application startup
  4. Provide default values where appropriate
  5. Document all required variables in your README
  6. Rotate keys regularly and especially after team member departures

Part 2: Optimizing the React App for Production (Building)

What is a Production Build?

A production build is an optimized version of your application, transformed and minified for better performance in a live environment. It involves several processes that make your app faster and smaller.

Key Optimization Techniques

1. Minification

Minification removes unnecessary characters from code without changing functionality:

// Development code (readable)
function calculateTotal(price, tax) {
  const taxAmount = price * tax;
  const total = price + taxAmount;
  return total;
}

// Minified production code
function calculateTotal(a,b){return a+(a*b);}

3. Tree Shaking

Tree shaking eliminates dead code (unused exports) from the final bundle:

// math.js - We export multiple functions
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;
export const multiply = (a, b) => a * b;
export const divide = (a, b) => a / b;

// App.js - We only use add and multiply
import { add, multiply } from './math';
// subtract and divide will be "shaken out" (removed) from the final bundle

4. Code Splitting

Code splitting allows you to split your bundle into smaller chunks loaded on demand:

// Without code splitting - entire app loads at once
import Dashboard from './pages/Dashboard';
import Settings from './pages/Settings';
import Profile from './pages/Profile';

// With code splitting - pages load only when needed
import React, { lazy, Suspense } from 'react';

const Dashboard = lazy(() => import('./pages/Dashboard'));
const Settings = lazy(() => import('./pages/Settings'));
const Profile = lazy(() => import('./pages/Profile'));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/settings" element={<Settings />} />
        <Route path="/profile" element={<Profile />} />
      </Routes>
    </Suspense>
  );
}

Part 3: Choosing a Hosting Platform

Overview of Popular Hosting Platforms for React Apps

1. Vercel

Best for: Next.js apps and static React sites
Pricing: Free tier available, paid plans start at $20/month

Key Features:

  • Automatic deployments from Git
  • Built-in CI/CD
  • Automatic SSL certificates
  • Edge Network for global distribution
  • Preview deployments for every PR
  • Serverless functions support

Getting Started with Vercel:

# 1. Install Vercel CLI
npm i -g vercel

# 2. Deploy from terminal
vercel

# 3. Follow the prompts
# - Set up and deploy: Y
# - Which scope: (select your account)
# - Link to existing project: N
# - Project name: my-react-app
# - Directory: ./dist (or ./build)

# 4. For production deployment
vercel --prod

Or deploy via Git:
1. Push code to GitHub/GitLab/Bitbucket
2. Import project at vercel.com
3. Automatic deployments on every push

2. Netlify

Best for: Static sites and JAMstack applications
Pricing: Free tier available, paid plans start at $19/month

Key Features:

  • Git-based workflow
  • Continuous deployment
  • Built-in form handling
  • Serverless functions
  • Split testing
  • Asset optimization

Getting Started with Netlify:

# 1. Install Netlify CLI
npm i -g netlify-cli

# 2. Build your project
npm run build

# 3. Deploy
netlify deploy

# 4. For production
netlify deploy --prod

Or deploy via Netlify Drop:
1. Run npm run build
2. Go to app.netlify.com/drop
3. Drag and drop your dist or build folder

3. GitHub Pages

Best for: Simple static sites, project documentation
Pricing: Free for public repositories

Key Features:

  • Free hosting
  • Directly integrated with GitHub
  • Custom domain support
  • Automatic HTTPS

Deploying to GitHub Pages:

# 1. Install gh-pages package
npm install --save-dev gh-pages

# 2. Add to package.json
{
  "homepage": "https://username.github.io/repo-name",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d build"
  }
}

# 3. Deploy
npm run deploy

4. Firebase Hosting

Best for: Full-stack apps with Firebase backend
Pricing: Free tier (10GB storage, 360MB data transfer/day)

Key Features:

  • Global CDN
  • One-click rollbacks
  • Integrated with Firebase services
  • Custom domains with SSL

Deploying to Firebase:

# 1. Install Firebase CLI
npm i -g firebase-tools

# 2. Login
firebase login

# 3. Initialize project
firebase init hosting

# 4. Build and deploy
npm run build
firebase deploy

5. AWS Amplify

Best for: Enterprise applications requiring AWS services
Pricing: Pay-as-you-go, free tier available

Key Features:

  • Full-stack deployment
  • Built-in CI/CD
  • Authentication integration
  • GraphQL API support
  • Scalable infrastructure

Step-by-Step Deployment Guide (Using Netlify as Example)

Step 1: Prepare Your Project

# Build your project
npm run build

# Test the build locally
npx serve -s dist

Step 2: Create a netlify.toml File

[build]
  command = "npm run build"
  publish = "dist"

[build.environment]
  NODE_VERSION = "18"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

Step 3: Deploy via Git

  1. Push code to GitHub
  2. Log in to Netlify
  3. Click "New site from Git"
  4. Select your repository
  5. Configure build settings: Build command: npm run build, Publish directory: dist
  6. Click "Deploy site"

Step 4: Configure Environment Variables

  1. Go to Site settings > Environment variables
  2. Add your production variables:
VITE_API_URL = https://api.production.com
VITE_GOOGLE_MAPS_KEY = prod_key_here

Step 5: Set Up Custom Domain

  1. Go to Domain settings
  2. Add custom domain
  3. Update DNS records
  4. Wait for SSL certificate generation

Step 6: Enable Additional Features

  • Enable form detection
  • Set up split testing
  • Configure redirects
  • Set up password protection (if needed)

Post-Deployment Checklist

  • [ ] Test all features in production
  • [ ] Verify environment variables are working
  • [ ] Check SSL certificate is valid
  • [ ] Test on multiple devices/browsers
  • [ ] Monitor performance metrics
  • [ ] Set up error tracking (Sentry, LogRocket)
  • [ ] Configure analytics (Google Analytics, Plausible)
  • [ ] Test backup and rollback procedures
  • [ ] Document deployment process
  • [ ] Set up monitoring alerts
