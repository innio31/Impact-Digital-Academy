Teaching Note: User Authentication (Simplified)

Overview for Instructors

This note provides guidance on teaching user authentication in a simplified manner, appropriate for a foundational web development course. Two approaches are presented: Firebase Authentication (a real backend service) and localStorage simulation (a mock approach for learning concepts). Both methods allow students to understand authentication flows without the complexity of building a full backend authentication system.

Part 1: Core Concepts to Cover

Before diving into implementation, ensure students understand:

Concept Description
Authentication vs. Authorization Authentication = verifying identity (who you are); Authorization = determining access (what you can do)

Part 2: Approach 1 - Firebase Authentication (Recommended for Real Projects)

Firebase Authentication provides a complete, production-ready solution with minimal setup. It handles security best practices automatically.

2.1 Setting Up Firebase

Step 1: Create a Firebase Project
1. Go to Firebase Console
2. Click "Add project" and follow the wizard
3. Register a web app to get configuration credentials

Step 2: Enable Email/Password Sign-In
1. In Firebase Console, navigate to "Authentication" â†’ "Sign-in method"
2. Enable "Email/Password" provider and save

Step 3: Install Firebase in Your React Project

npm install firebase

Step 4: Create Firebase Configuration File

// src/firebase.js
import { initializeApp } from "firebase/app";
import { getAuth } from "firebase/auth";

const firebaseConfig = {
  apiKey: "your-api-key",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
  // ... other config values
};

const app = initializeApp(firebaseConfig);
export const auth = getAuth(app);

2.2 Core Firebase Authentication Methods

Method Purpose
createUserWithEmailAndPassword(auth, email, password) Sign up new user
signInWithEmailAndPassword(auth, email, password) Sign in existing user
signOut(auth) Log out user
onAuthStateChanged(auth, callback) Listen to auth state changes

2.3 Sign-Up Implementation

import { createUserWithEmailAndPassword } from "firebase/auth";
import { auth } from "./firebase";

const handleSignUp = async (email, password) => {
  try {
    const userCredential = await createUserWithEmailAndPassword(auth, email, password);
    // User successfully signed up
    console.log("Signed up:", userCredential.user);
  } catch (error) {
    // Handle errors
    if (error.code === 'auth/email-already-in-use') {
      console.log("Email already registered");
    } else if (error.code === 'auth/weak-password') {
      console.log("Password too weak");
    }
  }
};

2.4 Sign-In Implementation

import { signInWithEmailAndPassword } from "firebase/auth";
import { auth } from "./firebase";

const handleSignIn = async (email, password) => {
  try {
    const userCredential = await signInWithEmailAndPassword(auth, email, password);
    // User successfully signed in
    console.log("Signed in:", userCredential.user);
  } catch (error) {
    if (error.code === 'auth/user-not-found') {
      console.log("No user found with this email");
    } else if (error.code === 'auth/wrong-password') {
      console.log("Incorrect password");
    }
  }
};

2.5 Managing Authentication State with React Context

For a complete React integration, use Context to provide auth state throughout the app :

// src/contexts/AuthContext.jsx
import { createContext, useContext, useState, useEffect } from 'react';
import { auth } from '../firebase';
import { 
  signInWithEmailAndPassword, 
  createUserWithEmailAndPassword,
  signOut,
  onAuthStateChanged 
} from 'firebase/auth';

const AuthContext = createContext();

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const unsubscribe = onAuthStateChanged(auth, (user) => {
      setUser(user);
      setLoading(false);
    });
    return unsubscribe;
  }, []);

  const login = (email, password) => {
    return signInWithEmailAndPassword(auth, email, password);
  };

  const signup = (email, password) => {
    return createUserWithEmailAndPassword(auth, email, password);
  };

  const logout = () => {
    return signOut(auth);
  };

  return (
    <AuthContext.Provider value={{ user, login, signup, logout }}>
      {!loading && children}
    </AuthContext.Provider>
  );
}

export const useAuth = () => useContext(AuthContext);

2.6 Protecting Routes

// src/components/ProtectedRoute.jsx
import { Navigate } from 'react-router-dom';
import { useAuth } from '../contexts/AuthContext';

export function ProtectedRoute({ children }) {
  const { user } = useAuth();
  
  if (!user) {
    return <Navigate to="/login" />;
  }
  
  return children;
}

Part 3: Approach 2 - Simulated Authentication with localStorage

For learning purposes or prototypes, you can simulate authentication using browser localStorage. Important: This approach is not secure for production as it stores credentials in plain text .

3.1 How It Works

Component Purpose
localStorage.setItem() Save user data or session token
localStorage.getItem() Retrieve stored session
localStorage.removeItem() Clear session on logout

3.2 User Registration (Simulated)

// Simulated sign-up
const handleSignUp = (email, password) => {
  // Get existing users or initialize empty array
  const users = JSON.parse(localStorage.getItem('users')) || [];
  
  // Check if user exists
  const userExists = users.some(user => user.email === email);
  
  if (userExists) {
    alert('Email already registered');
    return false;
  }
  
  // Add new user (NEVER store plain passwords in real apps!)
  users.push({ email, password });
  localStorage.setItem('users', JSON.stringify(users));
  
  // Auto-login after signup
  localStorage.setItem('currentUser', JSON.stringify({ email }));
  
  return true;
};

3.3 User Login (Simulated)

const handleLogin = (email, password) => {
  const users = JSON.parse(localStorage.getItem('users')) || [];
  
  // Find matching user
  const user = users.find(u => u.email === email && u.password === password);
  
  if (user) {
    // Create session
    localStorage.setItem('currentUser', JSON.stringify({ email }));
    return true;
  } else {
    alert('Invalid credentials');
    return false;
  }
};

3.4 Check Authentication Status

const getCurrentUser = () => {
  const userStr = localStorage.getItem('currentUser');
  return userStr ? JSON.parse(userStr) : null;
};

const isAuthenticated = () => {
  return !!localStorage.getItem('currentUser');
};

3.5 Logout (Simulated)

const handleLogout = () => {
  localStorage.removeItem('currentUser');
  // Redirect to login page
};

3.6 Complete Example Component

import { useState } from 'react';
import { useNavigate } from 'react-router-dom';

export function LoginPage() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const navigate = useNavigate();

  const handleSubmit = (e) => {
    e.preventDefault();
    
    // Simulated login
    const users = JSON.parse(localStorage.getItem('users')) || [];
    const user = users.find(u => u.email === email && u.password === password);
    
    if (user) {
      localStorage.setItem('currentUser', JSON.stringify({ email }));
      navigate('/dashboard');
    } else {
      alert('Invalid login');
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input 
        type="email" 
        value={email} 
        onChange={(e) => setEmail(e.target.value)} 
        required 
      />
      <input 
        type="password" 
        value={password} 
        onChange={(e) => setPassword(e.target.value)} 
        required 
      />
      <button type="submit">Login</button>
    </form>
  );
}

Part 4: Teaching Tips

4.1 Comparison of Approaches

Aspect Firebase Auth localStorage Simulation
Real-world use Yes, production-ready No, educational only
Setup complexity Medium Very low
Security High (handles hashing, etc.) None (plain text)
Persistence Automatic Manual
Social logins Supported Not available

4.2 Suggested Progression

  1. Start with localStorage simulation to teach authentication concepts without external dependencies
  2. Demonstrate the security flaws (show how passwords are visible)
  3. Introduce Firebase as the "real" solution that fixes these problems
  4. Build a complete auth flow with Firebase in a mini-project

4.3 Common Student Questions

Q: Why can't we just use localStorage for real apps?
A: localStorage is vulnerable to XSS attacks, stores data in plain text, and provides no real security. Firebase handles hashing, salting, and secure token management automatically .

Q: What happens when the page refreshes?
A: With Firebase, onAuthStateChanged fires and restores the user state automatically. With localStorage, you need to check localStorage.getItem() when the app loads .

Q: Can I use both approaches?
A: For learning, yes. Start with simulation to understand the flow, then transition to Firebase.
