üîó 1. Understanding URLs (Uniform Resource Locators)

A URL (web address) is the unique identifier for a resource on the internet. Think of it as a digital street address. Understanding its structure helps you navigate safely and identify legitimate websites.

1.1 The anatomy of a URL

https://www.example-inc.co.uk:443/blog/article?page=3&sort=recent#comments
Component Example Description
Scheme (protocol) https:// Protocol to access resource: http, https, ftp, mailto. https = secure
Subdomain www Subdivision of main domain (blog, mail, shop).
Domain name example-inc Unique human-readable name of the website.
Top-level domain (TLD) .co.uk Suffix: .com, .org, .gov, .edu or country-code (.uk, .ca).
Port :443 Technical gate (default 80 for http, 443 for https). Rarely visible.
Path /blog/article Specific file/directory location on server.
Query string (parameters) ?page=3&sort=recent Key=value pairs after ?, separated by &.
Fragment (anchor) #comments Jumps to a specific section inside the page.

1.2 Why understanding URLs matters

  • Security: Spot phishing: http://www.amazon.security-update.com ‚Üí domain = security-update.com (not amazon.com).
  • Navigation: Guess company website www.[company].com, support subdomain support.[company].com.
  • Information: Modify path or query to browse efficiently.
‚ö° Quick check: Always look for https:// and the correct domain name before entering passwords.
