üìå Part 1: Understanding Cloud Computing

1.1 What is Cloud Computing?

Imagine if, instead of keeping all your files and software on your computer's hard drive, you could access them from anywhere, using any device with an internet connection. That is the essence of cloud computing. It is like renting computing power and storage from a massive data center instead of owning and maintaining everything yourself.

Formally defined by the National Institute of Standards and Technology (NIST), cloud computing is a model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources (e.g., networks, servers, storage, applications, and services) that can be rapidly provisioned and released with minimal management effort or service provider interaction.

In simple terms, the "cloud" refers to servers that are accessed over the internet, and the software and databases that run on those servers.

1.2 Essential Characteristics of Cloud Computing

  1. On-Demand Self-Service: Provision capabilities automatically without human interaction.
  2. Broad Network Access: Available over the network via mobiles, laptops, workstations.
  3. Resource Pooling: Multi‚Äëtenant model, dynamically assigned resources.
  4. Rapid Elasticity: Scale rapidly outward/inward according to demand.
  5. Measured Service: Metering capability ‚Äì pay only for what you use.

1.3 Cloud Service Models ("Spend as You Grow")

IaaS (Infrastructure as a Service)
Virtual servers, storage, networks. You manage OS/apps/data.
Examples: AWS, Azure, Google Compute Engine
PaaS (Platform as a Service)
Develop apps without managing infrastructure. Provider handles OS, middleware.
Examples: Google App Engine, Salesforce, AWS Elastic Beanstalk
SaaS (Software as a Service)
Ready‚Äëto‚Äëuse software over internet. Provider manages everything.
Examples: Google Drive, Microsoft 365, Gmail

1.4 Cloud Deployment Models

  • Public cloud: Owned by third‚Äëparty provider, shared hardware.
  • Private cloud: Exclusive use by one organisation (on‚Äësite or hosted).
  • Hybrid cloud: Combines private and public for flexibility.
